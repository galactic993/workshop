# s1.5-reference-spec スキル

## 概要
画面設計書の参照仕様シートに記載されたテーブル名・カラム名が、テーブル定義書に実在するか検証する。

## 「リレーション」と他スキルとの役割分担

次は **別のチェック**として切り分けること。

| 観点 | 本スキル（s1.5-reference-spec） | その他 |
|------|--------------------------------|--------|
| 参照仕様に書かれた **テーブル論理名・項目論理名**が、渡されたテーブル定義書の **テーブル一覧／各シート**に存在するか | 対象 | — |
| テーブル定義書 **内部の FK**（備考列の参照先テーブル・カラム） | 対象外 | `run-all-check.sh` 実行時の **c1-fk-reference-target-exists** が該当 |
| 参照仕様に書かれた **結合・取得の業務意味**（SQL が正しいか、JOIN キーが妥当か） | 対象外 | ルール化する場合は別スキルまたは人間レビュー |

## テーブル定義書の冊数（辞書の範囲）

- `run-all-check.sh` は **テーブル定義書を1パス**受け取る。参照仕様が **共・売・編の別ブック**に分かれているマスタを指す場合、**渡した1冊だけ**を辞書にすると、他ブックにしかないテーブル名が **未定義参照と誤判定**されうる。
- レポートでは、**参照先が別区分ブックにある可能性**がある場合は **WARN** とし、**実際に検証したブック名／パス**を明記する。

## 使い方

`shared-agent-skills` では `scripts/run-all-check.sh` に画面設計書・テーブル定義書を渡すと、本スキルを含む s1.5 プロンプトが並列実行されます。

単体で使う場合は本ディレクトリの `SKILL.md` を `~/.claude/skills/` に置き、Claude / Claude Code で Excel を読み取れるように指示してください。

## チェック内容
1. **手元で利用可能なテーブル定義書**（実行時に渡されたファイル）から、テーブル論理名・カラム論理名の辞書を構築する。共・売・編の **3冊すべて**がある場合はマージした辞書を使ってよい。
2. 参照仕様シートから◆セクションおよび `<抽出テーブル>` / `<抽出項目>` 付近の **テーブル論理名・項目論理名**を抽出する。
3. テーブル名・カラム名をそれぞれ辞書と突合する。
4. 存在しないテーブル/カラムへの参照を検出する。

## レビュー上の判定
- **PASS**: 参照先がテーブル定義書に存在
- **WARN/FAIL**: 未定義参照または要確認
