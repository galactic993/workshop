# C1: FK参照先テーブル/カラム存在チェック（横断チェック）

**重要度:** error
**対象シート:** テーブル定義書の全テーブルシート

## プロンプト

```
テーブル定義書の全シートを対象に、FK（外部キー）参照の整合性をチェックしてください。

【各テーブルシートの構造】
- ヘッダー：B列=No., E列=論理名, N列=物理名, W列=データ型, AC列=桁数, AF列=Not Null, AI列=PK, AK列=UK, AM列=FK, AO列=Default, AT列=備考
- FK参照先はAT列（備考）に「テーブル名.カラム名　FK制約cascade」のような形式で記述

チェック内容：
1. 全シートを走査し、AT列（備考）に「FK」を含む行を抽出
2. FK参照先のテーブル名とカラム名を解析
3. 参照先テーブルのシートが存在するか確認
4. 参照先カラムがそのテーブルのN列に存在するか確認

結果を以下の形式で出力してください：
| シート名 | 行番号 | カラム名 | FK参照先 | テーブル存在 | カラム存在 | チェック結果 |
|----------|--------|----------|----------|--------------|------------|--------------|
例：| 注文明細 | 8 | quot_id | 見積.quot_id | あり | あり | OK |
    | 注文明細 | 10 | cust_cd | 顧客.cust_code | あり | なし | エラー: 参照先カラムなし |
    | 注文明細 | 12 | prod_id | 商品.prod_id | なし | - | エラー: 参照先テーブルなし |
```

## 説明

このプロンプトは、テーブル定義書内のすべてのFK（外部キー）定義をチェックします。

- **対象**: すべてのテーブルシートのFK定義（備考列）
- **チェック内容**:
  - FK参照先のテーブルが存在するか
  - FK参照先のカラムが存在するか
- **重要性**: データベースの整合性を確保するための必須チェック

## 使用上の注意

- テーブル定義書のすべてのシートが対象となります
- FK参照先の記述形式が統一されていることを確認してください（「テーブル名.カラム名」形式）
- テーブルシート名が物理テーブル名と一致していることを確認してください
- 「表紙」や「変更履歴」などのシートは自動的に除外されます
- FK参照先が存在しない場合は、テーブル定義を修正するか、FK定義を削除してください
