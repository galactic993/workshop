# B1: DB登録値のフィールド名存在チェック

**重要度:** error
**対象シート:** 画面設計書の「DB登録値」シート + テーブル定義書の各テーブルシート

## プロンプト

```
【前提】画面設計書とテーブル定義書の両方を開いてください。

画面設計書の「DB登録値」シートとテーブル定義書をクロスチェックします。

【DB登録値シートの構造】
- テーブルごとにセクション分けされています
- ヘッダー：B列=No., G列=項目名, O列=フィールド名, Y列=その他条件, AN列=登録値

【テーブル定義書の構造】
- 各テーブルのシート名がテーブル名
- 各シート内：B列=No., E列=論理名, N列=物理名（これがフィールド名）

チェック内容：
1. DB登録値シートの各セクションからテーブル名とフィールド名（O列）を抽出
2. 対応するテーブル定義書シートのN列にそのフィールド名が存在するか確認
3. 存在しないフィールド名をリストアップ

結果を以下の形式で出力してください：
| DB登録値セクション | 行番号 | 項目名 | 入力フィールド名 | チェック結果 |
|--------------------|--------|--------|------------------|--------------|
例：| 顧客テーブル | 15 | 氏名 | customer_nm | OK |
    | 顧客テーブル | 18 | 部署 | dept_code | エラー: テーブル定義書に存在しません |
```

## 説明

このプロンプトは、2つのファイルをクロスチェックするB系プロンプトです。

- **画面設計書側**: DB登録値シートに定義されているフィールド名
- **テーブル定義書側**: 実装対象のテーブル定義
- **チェック内容**: 画面で使用されるフィールド名がテーブル定義書に存在するか確認

## 使用上の注意

- 必ず両方のファイルを開いてから実行してください
- DB登録値シートのテーブルセクションを正確に識別することが重要です
- テーブル定義書のシート名がテーブル名と一致していることを確認してください
- フィールド名（物理名）は大文字小文字が正確に一致している必要があります
