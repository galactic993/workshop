# s1.5-message-format スキル

## 概要
画面設計書のメッセージ一覧シートを対象に、エラー/サクセスメッセージが定義済みフォーマットパターンに準拠しているかチェックする。

## 使い方

`shared-agent-skills` では `scripts/run-all-check.sh` に画面設計書・テーブル定義書を渡すと、本スキルを含む s1.5 プロンプトが並列実行されます。

単体で使う場合は本ディレクトリの `SKILL.md` を `~/.claude/skills/` に置き、Claude / Claude Code で Excel を読み取れるように指示してください。

## フォーマットルール（6パターン）
| パターンID | 区分 | 例 |
|---|---|---|
| MSG-AUTH | エラー | アクセス権限がありません |
| MSG-FETCH-FAIL | エラー | XXXの取得に失敗しました。時間を空けて再度お試しください |
| MSG-PROC-FAIL | エラー | XXXに失敗しました |
| MSG-INPUT-ERR | エラー | 入力内容に誤りがあります。各項目をご確認ください |
| MSG-NOT-FOUND | サクセス | 該当するXXXが見つかりません |
| MSG-SUCCESS | サクセス | XXXしました |

## 照合する列の特定（重要）

- パターン照合の対象は、**ユーザー向けに表示される本文**が書かれている列であること。多くの設計書では **「条件」列は説明用**で、**短い定型文（アラート／メッセージ本文）が別列**にある。
- **列位置の決め打ち（例: 常にG列）をしない**。メッセージ一覧シートで **ヘッダー行を探し、「メッセージ」「アラート」等の列名**を特定してから、その列のセルだけを 6 パターンと照合する。
- 「条件」列のみを見て逸脱と判定しないこと。**本文列**と **区分（エラー／サクセス）列**をセットで確認する。

## レビュー上の判定
- **PASS**: 定義済みパターンに準拠
- **WARN/FAIL**: 逸脱または要確認
