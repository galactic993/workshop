# A1: イベント番号の連番性チェック

**重要度:** warning
**対象シート:** イベント記述書

## プロンプト

```
イベント記述書シートを確認してください。
- ヘッダーは5行目にあります（B列=EVENT No., G列=トリガー, U列=区分, Z列=処理内容）
- EVENT No.はB列に「EVENT0001」「EVENT0002」のような形式で入力されています

以下をチェックしてください：
1. すべてのEVENT番号を抽出し、数値部分の連番を確認
2. 抜け番（例：EVENT0003の次がEVENT0005）を検出
3. 重複番号を検出

結果を以下の形式で出力してください：
| 行番号 | EVENT番号 | チェック結果 | 詳細 |
|--------|-----------|--------------|------|
例：| 6 | EVENT0001 | OK | - |
    | 8 | EVENT0005 | 警告 | EVENT0003, EVENT0004 が欠落 |
```

## 説明

このプロンプトは、画面設計書内のイベント記述書シートで、イベント番号が正しく連番になっているかを確認します。

- **抜け番の検出**: EVENT0001 → EVENT0002 → EVENT0004 のように欠落があれば警告
- **重複の検出**: 同じイベント番号が複数行に登録されていないか確認
- **形式の確認**: EVENT + 4桁の数値という形式が統一されているか確認

## 使用上の注意

- イベント記述書のヘッダーが5行目にあることを確認してから実行してください
- ヘッダーが異なる行にある場合は、プロンプト内の「5行目」を修正してください
- 空行を含む場合は自動的に除外されます
