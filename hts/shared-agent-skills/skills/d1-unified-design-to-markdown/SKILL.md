# D1: 統合設計書Markdown（画面設計書 + テーブル定義書）

**対象ファイル:** 画面設計書 + テーブル定義書（両方を開いた状態で実行）

## 説明

画面設計書とテーブル定義書の両方を読み取り、関連テーブルの情報を画面設計書側に埋め込む形で **1つの統合 Markdown** として出力します。

## プロンプト

```
現在開いている画面設計書とテーブル定義書の両方を読み取り、1つの統合Markdownドキュメントとして出力してください。

■ 出力構造

# {画面名}（例: 制作見積書作成<トップ画面>）

## 1. 基本情報
- 表紙の情報（システム名、サブシステム名、画面名、作成日、変更日）

## 2. 画面概要
- 画面概要シートの説明文、利用権限一覧、I/O情報

## 3. イベント定義
- イベント記述書シートの全EVENTをh3見出し付きで出力

## 4. 項目定義
- 項目記述書シートの全項目をMarkdownテーブルで出力

## 5. 参照仕様
- 参照仕様シートの内容を構造化して出力

## 6. DB登録値
- DB登録値シートの内容をテーブルごとにMarkdownテーブルで出力
- 【追加】各フィールドの横に、テーブル定義書から取得したデータ型とNot Null制約を補記する
  - 例: `prod_quot_status` (CHAR(2), NOT NULL, Default: '00:未着手')

## 7. 表示切替項目
- 表示切替項目シートの内容を条件別に出力

## 8. メッセージ一覧
- メッセージ一覧シートの全メッセージをMarkdownテーブルで出力

## 9. 関連テーブル定義
- DB登録値シートと参照仕様シートで参照されているテーブルのみをテーブル定義書から抽出
- 全テーブルではなく、この画面に関連するテーブルだけを出力
- 各テーブルのカラム定義をMarkdownテーブルで出力
  - 列: No. | 論理名 | 物理名 | データ型 | 桁数 | NOT NULL | PK | FK | Default | 備考
- 「●」は ✔ に変換。空セルは - で埋める

すべての出力をMarkdownのコードブロックなしで、そのままMarkdown文書として出力してください。
```

## 使用上の注意

- **必須**: 画面設計書とテーブル定義書の両方を開いた状態で実行してください
- 各ファイルのシート構成が標準的な構成であることを確認してください
- DB登録値と参照仕様に記載されているテーブルが自動的に抽出されます
- 関連テーブル以外は出力に含まれないため、**テーブル定義書の全シート一覧**が別途必要な場合は、テーブル定義書 Excel を直接参照してください

## 推奨ワークフロー

1. このプロンプトで統合 Markdown を生成する
2. 出力された Markdown をコピーし、`.md` ファイルとして保存
3. Claude Code などに読み込ませて開発作業に使用
