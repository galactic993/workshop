# s1.5-validation-consistency スキル

## 概要
画面設計書の項目記述書に記載された制御内容（必須/桁数制限等）と、メッセージ一覧のメッセージが対応しているかチェックする。

## 使い方

`shared-agent-skills` では `scripts/run-all-check.sh` に画面設計書・テーブル定義書を渡すと、本スキルを含む s1.5 プロンプトが並列実行されます。

単体で使う場合は本ディレクトリの `SKILL.md` を `~/.claude/skills/` に置き、Claude / Claude Code で Excel を読み取れるように指示してください。

## チェック内容
1. 項目記述書で **ヘッダー行を検索**し、「制御内容」「メッセージ」「EVENT No.」等の列を特定する（列番号の決め打ちをしない）
2. 制御内容列から制御ルールを抽出する（必須、桁数、半角数字 等）
3. メッセージ一覧では **ヘッダーで「メッセージ」またはアラート本文列**を特定し、メッセージを抽出する
4. 入力チェック系制御ルールに対して対応するエラーメッセージがメッセージ一覧に存在するか確認する
5. 不足しているメッセージを検出する

**補足:** 項目記述書セル内の **文言フォーマットそのもの**だけを細かく監査する場合は、本スキルに加え **s1.5-message-format**（メッセージ一覧の定型文）と役割を分けて判断する。

## レビュー上の判定
- **PASS**: 制御ルールとメッセージの対応に問題なし
- **WARN/FAIL**: 不足・不整合または要確認
