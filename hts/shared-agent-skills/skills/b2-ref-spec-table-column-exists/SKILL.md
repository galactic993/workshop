# B2: 参照仕様のテーブル名.カラム名パターン存在チェック

**重要度:** warning
**対象シート:** 画面設計書の「参照仕様」シート + テーブル定義書

## プロンプト
```
【前提】画面設計書とテーブル定義書の両方を開いてください。

画面設計書の「参照仕様」シートとテーブル定義書をクロスチェックします。

【参照仕様シート】
- 自由形式のテキストで、データ参照元が記載されています
- 「テーブル名.カラム名」のパターンで記述されることが多いです（例：顧客.customer_id）

【テーブル定義書の構造】
- シート名 = テーブル名（物理名）
- 各シートのN列 = カラム名（物理名）

チェック内容：
1. 参照仕様シートから「テーブル名.カラム名」のパターンを抽出
   - 形式：英数字アンダースコア.英数字アンダースコア
2. 各パターンについて、テーブル定義書の対応シートとN列を確認
3. 存在しないテーブルまたはカラムをリストアップ

結果を以下の形式で出力してください：
| 抽出されたパターン | テーブル存在 | カラム存在 | チェック結果 |
|--------------------|--------------|------------|--------------|
例：| customer.customer_id | あり | あり | OK |
    | order.item_code | あり | なし | 警告: カラムが存在しません |
    | product_info.item_id | なし | - | 警告: テーブルが存在しません |
```

## 説明

このプロンプトは、参照仕様シートで記載されているデータ参照がテーブル定義書と一致しているかを確認します。

- **対象**: 参照仕様シートの自由形式テキストから抽出される「テーブル名.カラム名」パターン
- **チェック内容**: そのテーブルとカラムがテーブル定義書に実装されているか確認
- **重要性**: 画面の参照データソースが実装側と乖離していないか確認

## 使用上の注意

- 必ず両方のファイルを開いてから実行してください
- 参照仕様シートの形式が「テーブル名.カラム名」パターンを採用していることを確認してください
- テーブル定義書のシート名がテーブル名と一致していることを確認してください
- パターン抽出がうまくいかない場合は、参照仕様シートの記述形式を統一することを推奨します
