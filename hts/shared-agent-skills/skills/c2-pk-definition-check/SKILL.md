# C2: PK定義チェック

**重要度:** warning
**対象シート:** テーブル定義書の全テーブルシート + テーブル一覧シート

## プロンプト

```
テーブル定義書の全テーブルについて、主キー（PK）が定義されているかチェックしてください。

【テーブル一覧シート】
- ヘッダー：B列=No., E列=論理名, P列=物理名, AA列=種別, AG列=概要
- P列にテーブル名（物理名）がリストされています

【各テーブルシート】
- AI列=PK（主キーフラグ）
- AI列にチェックマークや○が入っている場合、そのカラムがPK

チェック内容：
1. テーブル一覧シートから全テーブル名を取得
2. 各テーブルシートを確認し、AI列（PK）に値がある行が存在するか
3. PKが定義されていないテーブルをリストアップ

結果を以下の形式で出力してください：
| テーブル名 | シート名 | PK定義あり | チェック結果 |
|------------|----------|------------|--------------|
例：| customer | 顧客 | あり（customer_id） | OK |
    | temp_data | 一時データ | なし | 警告: PK未定義 |
```

## 説明

このプロンプトは、テーブル定義書のすべてのテーブルが主キー（PK）を持つかどうかをチェックします。

- **対象**: テーブル一覧シートで定義されているすべてのテーブル
- **チェック内容**: 各テーブルが主キーを持つか確認
- **重要性**: テーブルの一意性を保証するために、ほとんどのテーブルにはPKが必要

## 使用上の注意

- テーブル一覧シートが存在することを確認してください
- テーブル一覧シートのヘッダーが記載されている行番号を確認してください
- 一時テーブルやサマリーテーブルなど、PKが不要なテーブルもあります
- PKが定義されていないテーブルが見つかった場合は、設計者に確認してください
- AI列（PK）のフラグ形式（○、×、Y、N等）が統一されていることを確認してください
