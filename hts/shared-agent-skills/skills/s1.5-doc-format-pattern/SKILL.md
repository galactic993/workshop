# s1.5-doc-format-pattern スキル

## 概要
複数の画面設計書を横断分析し、シート構成・ヘッダー位置・列構成・採番パターン・メッセージパターンの共通性と差異を検出する。

## 使い方

`shared-agent-skills` では `scripts/run-all-check.sh` に画面設計書・テーブル定義書を渡すと、本スキルを含む s1.5 プロンプトが並列実行されます。

単体で使う場合は本ディレクトリの `SKILL.md` を `~/.claude/skills/` に置き、Claude / Claude Code で Excel を読み取れるように指示してください。

## チェック内容

### 複数の画面設計書がある場合（本スキルの主用途）

1. 全画面設計書のシート構成を比較（共通シートの存在確認）
2. 各シートのヘッダー行位置・列数の一致/差異を検出
3. EVENT No 採番パターン（開始番号、形式）を分析
4. メッセージ文言をパターン分類し、頻度表を生成

### 画面設計書が1ファイルだけの場合

横断比較はできないため、次を **単体自己チェック**として行う。

1. **期待する共通シート**（下記）の有無を確認し、欠けがあれば **WARN**
2. 上記以外の **余分なシート**（例: メモ用 `Sheet1`）があれば **WARN**（削除必須ではないが、テンプレ逸脱として記録）
3. 当該ファイル内の EVENT／メッセージのパターンを要約する（頻度表は1冊分）

## 期待する共通シート
- 表紙, 変更履歴, 画面概要, イベント記述書, 項目記述書, 参照仕様, DB登録値, 表示切替項目, メッセージ一覧

## レビュー上の判定
- **PASS**: 横断比較または単体チェックで重大な不整合なし
- **WARN/FAIL**: 差異ありまたは要確認
