# s1.5-cowork-screen-capture-review

## 位置づけ

- `s1.5-screen-capture` は `run-session1-claude-p.sh` 経由で同じ観点を **`claude -p`** で実行する **プロンプトスキル**。
- 本スキルは **Claude Cowork / チャット上で同じ観点のレビューを手動オーケストレーションする**ための **プロンプトテンプレート**です。画像解析環境がない場合や、表記の解釈を人間が確認したい場合に使う。

## 基本プロンプト（コピー用）

次のブロックをそのまま Cowork に貼り、画像と項目記述書の抜粋を添付する。

```markdown
あなたは画面設計書レビュー担当です。次の2つを突き合わせてください。

【入力】
1. 画面概要シートに埋め込まれている画面キャプチャ画像（または添付画像）
2. 同じ設計書の「項目記述書」シートの内容（項目名が分かる範囲を貼付または要約）

【やること】
- キャプチャ上で読み取れる「ラベル」「ボタン文言」「リンク文言」をできるだけ列挙する
- 項目記述書の「項目名」一覧と照合し、次を分類する
  - 両方にある
  - キャプチャにだけある（設計書に記載がない／表記ゆれの可能性）
  - 項目記述書にだけある（画面に見えない／別画面の可能性）
- 表記ゆれ（全角半角、スペース、「：」の有無など）が疑われる場合は指摘する

【出力形式】
- 表形式（項目名、キャプチャ、項目記述書、判定、メモ）
- 最後に優先度の高い差分を3件以内で要約

不明な文字は推測せず「判読不能」と書いてください。
```

## 短い版（画像だけ先に渡すとき）

**1通目（画像のみ）:**

```markdown
この画面キャプチャから、フォームの項目ラベルとボタン名をすべて列挙し、JSON 配列風のリストで出力してください。説明文は不要です。
```

**2通目（項目記述書を渡すとき）:**

```markdown
上記リストと、次の項目記述書の項目名を突合し、不足・余分・表記ゆれを表で出してください。
```

## 精度を上げるコツ

- 画像は **解像度が高いもの**（Excel の埋め込みをそのままエクスポート、またはシートを PNG/PDF で切り出し）を渡す。
- 項目記述書は **スクリプトと同じ前提**で、**G列（7列目）の「項目名」** がチェック対象であることをプロンプトに書いておくとブレにくい。
- 社内の表記ルール（例: ボタンは「登録」固定、ラベルは「〇〇名」）があれば1行追記する。
