# 一括チェック（`run-all-check.sh`）

## 何をするか

[`run-all-check.sh`](run-all-check.sh) は、**画面設計書**と**テーブル定義書**のパスを渡すと、`claude -p` で複数の **SKILL.md プロンプト**を並列実行し、結果を `session1-claude-runs/<日時>/` にログ保存する。

- **s1.5 系**（`s1.5-doc-format-pattern` … `s1.5-screen-capture`）だけでなく、**a1–c3**（イベント連番、DB 登録、参照・FK 等）も同じバッチに含まれる（`SKIP_S15_PROMPTS=1` で s1.5 を省略可能）。
- 実行後、オプションで **コピーした Excel にレビュー結果を反映**する対話フロー（[`prompts/fix_workbooks.md`](prompts/fix_workbooks.md)）がある。

## 使い方

```bash
cd /path/to/shared-agent-skills
./scripts/run-all-check.sh /path/to/画面設計書.xlsx /path/to/テーブル定義書.xlsx
```

## テーブル定義書の冊数について

- スクリプトは **テーブル定義書を1ファイル**指定する。
- 参照仕様が **共・売・編の別ブック**に分かれているテーブルを指す場合、**渡した1冊だけ**を辞書にすると、他ブックにしかない論理名が **未定義参照と誤判定**されうる。
- **s1.5-reference-spec** のレポートでは、検証に使ったパスを明記し、別ブック参照の可能性は **WARN** とする運用を推奨する。

## 「リレーション／参照」の役割分担（s1.5 と a–c）

| 観点 | 主なスキル |
|------|------------|
| 参照仕様シートの **テーブル論理名・項目論理名**が、渡されたテーブル定義書に **存在するか** | **s1.5-reference-spec**（同バッチでは **b2** 系の思想と近い） |
| テーブル定義書 **内部の FK**（備考の参照先テーブル・カラム） | **c1-fk-reference-target-exists**（テーブル定義書シートを走査） |
| 参照仕様に書かれた **結合・取得の業務意味**（SQL が正しいか等） | **現状のスキル対象外**（別スキル or 人間レビュー） |

## s1.5 個別スキルの概要

| スキル | 目的 |
|--------|------|
| s1.5-doc-format-pattern | 複数設計書のフォーマット差分、または単体時は期待シート・余分シートの確認 |
| s1.5-event-no | EVENT No 欠番・重複、項目記述書との未参照、振り直し案 |
| s1.5-message-format | メッセージ一覧の本文が 6 パターンに沿うか |
| s1.5-validation-consistency | 項目記述書の制御内容とメッセージ一覧の対応 |
| s1.5-reference-spec | 参照仕様の表・項目がテーブル定義書に存在するか |
| s1.5-screen-capture | 画面概要の画像（またはテキスト）と項目記述書の突合 |

詳細は各 [`skills/s1.5-*/SKILL.md`](../skills/) を参照。
