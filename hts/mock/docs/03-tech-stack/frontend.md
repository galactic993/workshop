# フロントエンド技術スタック

## フレームワーク

### Next.js
- **バージョン**: 15.x (App Router)
- **用途**: Reactベースのフルスタックフレームワーク
- **採用理由**:
  - サーバーサイドレンダリング（SSR）対応
  - ファイルベースルーティング
  - 優れた開発体験
  - 大規模アプリケーションに適している

## コア言語

### TypeScript
- **バージョン**: 5.x
- **用途**: 型安全な開発
- **採用理由**:
  - 型安全性による品質向上
  - IDEサポートの充実
  - リファクタリングの容易さ
  - チーム開発での可読性向上

## 状態管理

### Zustand
- **用途**: グローバル状態管理
- **使用例**:
  - ユーザー情報
  - テーマ設定
  - UI状態
- **採用理由**:
  - シンプルで軽量
  - Reduxより学習コストが低い
  - TypeScriptサポートが優秀
  - Providerが不要

### TanStack Query (React Query)
- **バージョン**: 5.x
- **用途**: サーバー状態管理
- **使用例**:
  - APIデータのキャッシュ
  - データの自動再取得
  - 楽観的更新
- **採用理由**:
  - キャッシュ管理が自動化
  - ローディング状態の管理が簡単
  - パフォーマンス向上
  - デファクトスタンダード

## フォーム管理

### React Hook Form
- **バージョン**: 7.x
- **用途**: フォーム状態管理とバリデーション
- **採用理由**:
  - 高パフォーマンス（再レンダリング最小化）
  - zodとの統合が容易
  - シンプルなAPI
  - バリデーションが柔軟

### @hookform/resolvers
- **用途**: React Hook FormとZodの連携
- **採用理由**: zodバリデーターの統合に必須

## バリデーション

### Zod
- **バージョン**: 3.x
- **用途**: スキーマバリデーション
- **使用例**:
  - フォーム入力のバリデーション
  - APIレスポンスの型検証
  - 型安全性の確保
- **採用理由**:
  - TypeScriptファーストの設計
  - 型推論が優秀
  - エラーメッセージのカスタマイズが容易
  - React Hook Formとの相性が良い

## HTTP通信

### Axios
- **バージョン**: 1.x
- **用途**: HTTPクライアント
- **採用理由**:
  - インターセプター機能
  - リクエスト/レスポンスの変換が容易
  - タイムアウト設定
  - デファクトスタンダード

## スタイリング

### Tailwind CSS
- **バージョン**: 3.x
- **用途**: ユーティリティファーストCSS
- **採用理由**:
  - 高速な開発が可能
  - デザインの一貫性が保ちやすい
  - カスタマイズ性が高い
  - ビルドサイズの最適化

## UIコンポーネント（検討中）

以下のライブラリから選定予定：

### shadcn/ui
- Tailwind CSSベース
- コピー&ペースト型（依存関係が少ない）
- カスタマイズ性が高い

### Headless UI
- アクセシビリティに優れている
- Tailwindとの相性が良い

## アイコン

### Lucide React / Heroicons
- **用途**: アイコンライブラリ
- **採用理由**:
  - モダンなデザイン
  - Tree-shakingに対応
  - カスタマイズ可能

## 開発ツール

### ESLint
- **用途**: コード品質チェック
- **設定**: Next.js推奨設定 + カスタムルール

### Prettier
- **用途**: コードフォーマット
- **設定**: プロジェクト統一のフォーマットルール

### TypeScript ESLint
- **用途**: TypeScript向けLintルール
- **採用理由**: TypeScriptの型安全性を強化

## テスト

### Vitest
- **用途**: ユニットテスト・統合テスト
- **採用理由**:
  - Viteベースで高速
  - Jest互換のAPI
  - TypeScriptサポートが優秀

### @testing-library/react
- **用途**: Reactコンポーネントのテスト
- **採用理由**:
  - ユーザー視点のテストが書ける
  - デファクトスタンダード
  - アクセシビリティを考慮したテスト

### @testing-library/user-event
- **用途**: ユーザーインタラクションのシミュレーション
- **採用理由**: より実際のユーザー操作に近いテストが可能

## ユーティリティ

### date-fns
- **用途**: 日付操作
- **採用理由**:
  - 軽量
  - Tree-shakingに対応
  - 日本語ロケール対応

## ディレクトリ構成

```
frontend/
├── app/                    # Next.js App Router
│   ├── (auth)/            # 認証関連ページ
│   ├── (dashboard)/       # ダッシュボード
│   └── api/               # API Routes
├── components/            # Reactコンポーネント
│   ├── ui/               # UIコンポーネント
│   └── features/         # 機能別コンポーネント
├── hooks/                # カスタムフック
├── lib/                  # ユーティリティ
│   ├── api.ts           # Axios設定
│   └── utils.ts         # 汎用関数
├── stores/              # Zustandストア
├── types/               # TypeScript型定義
├── styles/              # グローバルスタイル
└── __tests__/          # テストファイル
```
