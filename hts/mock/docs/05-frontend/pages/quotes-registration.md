# 見積登録・更新ページ仕様書

## 概要

見積の新規登録・詳細表示・更新を行う専用ページ。

- **URL**: `/sales/quotes/[id]`
  - `/sales/quotes/new` → 新規登録モード
  - `/sales/quotes/{数字}` → 詳細・更新モード
- **権限**: `sales.quotes.*` または アクセス区分 `00`
- **バージョン**: 1.0.2

## 権限チェック

`useAuthGuard`フックにより自動的に処理される。

| 条件 | 動作 |
|------|------|
| 未認証 | 「ログインしてください」エラー表示後、ホーム画面（`/`）へリダイレクト |
| 権限なし | 「アクセス権限がありません」エラー表示後、ホーム画面（`/`）へリダイレクト |
| 権限あり | 通常表示 |

## エラーハンドリング

### サーバーエラー表示

センター一覧取得などのAPI呼び出しでサーバーエラーが発生した場合、画面上部に`ErrorMessageList`コンポーネントでエラーメッセージを表示する。

| エラー種別 | 表示メッセージ |
|-----------|---------------|
| センター取得エラー | センターの取得に失敗しました。時間を空けて再度お試しください |
| 得意先検索エラー | 得意先の取得に失敗しました。時間を空けて再度お試しください |

### エラー表示位置

- **新規登録モード**: フォームヘッダーの下、入力フィールドの上に表示
- **詳細・更新モード**: 基本情報表示部分の下、編集セクションの上に表示

### 技術的な注意事項

- 画面遷移時（コンポーネントマウント時）にエラー状態をクリアする
- エラーメッセージは追記せず、直接設定することでエラー重複を防止

```tsx
// エラー状態の管理
const [errors, setErrors] = useState<string[]>([]);

// データ取得時のエラーハンドリング
useEffect(() => {
  const fetchCenters = async () => {
    try {
      setCentersLoading(true);
      setErrors([]); // エラーをクリア
      const response = await getQuotCenters();
      if (response.success) {
        setCenters(response.centers);
      } else {
        setErrors(['センターの取得に失敗しました。時間を空けて再度お試しください']);
      }
    } catch (error) {
      console.error('センター取得エラー:', error);
      setErrors(['センターの取得に失敗しました。時間を空けて再度お試しください']);
    } finally {
      setCentersLoading(false);
    }
  };
  fetchCenters();
}, []);
```

### 表示項目

| 項目 | 表示形式 | 表示内容 |
|------|---------|---------|
| 【部署】 | ラベル + 値（横並び） | 部署コード + 部署名 |
| 【担当者】 | ラベル + 値（横並び） | 社員名 |

ラベル幅: `w-20`（5rem）

### 入力項目

入力項目はすべて横並びレイアウト（ラベル幅: `w-28`）。見積概要以外は入力欄幅 `w-1/2`。

| 項目名 | 入力形式 | 必須 | 入力欄幅 | バリデーション |
|--------|----------|------|---------|----------------|
| 得意先 | テキスト + 選択ボタン | - | w-1/2 | - |
| 得意先名 | テキスト | - | w-1/2 | 120文字以内 |
| 見積件名 | テキスト | - | w-1/2 | 50文字以内 |
| 品名 | テキスト | - | w-1/2 | 50文字以内 |
| 見積概要 | テキストエリア（4行） | - | flex-1 | 2000文字以内 |
| 提出方法 | ドロップダウン | ○ | w-1/2 | 初期値: 未定 |
| 参考資料 | テキスト | - | w-1/2 | 255文字以内 |
| 主管センター | ドロップダウン | - | w-1/2 | - |
| 伝達事項 | テキストエリア（4行） | - | flex-1 | 2000文字以内 |

※ 得意先が「諸口」(得意先コード: 33900)の場合のみ得意先名フィールドを表示

### バリデーションエラーメッセージ（新規登録時）

| 項目 | エラー種別 | メッセージ |
|------|-----------|-----------|
| 得意先名 | 桁数 | 120文字以内で入力してください |
| 見積件名 | 桁数 | 50文字以内で入力してください |
| 品名 | 桁数 | 50文字以内で入力してください |
| 見積概要 | 桁数 | 2000文字以内で入力してください |
| 提出方法 | 必須 | 提出方法を選択してください |
| 参考資料 | 桁数 | 255文字以内で入力してください |
| 伝達事項 | 桁数 | 2000文字以内で入力してください |

※ 提出方法は必須（初期値が「未定」のため通常はエラーにならない）

### 得意先選択の対象範囲

新規登録モーダル内の得意先選択は、アクセス権限に関わらず以下の一律ロジックで決定される。

1. ログインユーザーの所属部署を取得
2. 所属部署からセンターを特定
   - 所属部署がセンター → そのセンター
   - 所属部署がチーム → 親センター（`center_id`）
3. センターの組織別部署コード（`department_section_cd`）から部署コードIDを取得
4. その部署コードに紐づく部署別得意先（`customer_section_cd`）を取得

**注意**: 検索フォームの得意先選択とは対象範囲が異なる。

### 主管センター選択肢

- 編集・印刷センター一覧を表示（`/api/centers/quot`）
- 組織区分が「20:編集」または「30:印刷」のセンターのみ選択可能

### 新規登録用得意先検索API

検索フォームの得意先検索APIとは別のエンドポイントを使用する。

#### サジェスト検索

- URL: `GET /api/quotes/customers/suggest-for-create`
- パラメータ: `query` (検索クエリ)
- 対象: ログインユーザーの所属センターに紐づく部署別得意先

#### ダイアログ検索

- URL: `GET /api/quotes/customers/search-for-create`
- パラメータ: `customer_cd`, `customer_name`
- 対象: ログインユーザーの所属センターに紐づく部署別得意先

### フッターボタン（新規登録モード）

| ボタン | 位置 | 色 | 動作 |
|--------|------|-----|------|
| 一覧に戻る | フッター左 | 白（ボーダー） | 未保存の変更がある場合は確認ダイアログを表示、なければ一覧画面へ遷移 |
| 登録 | フッター右 | 青 | 登録確認ダイアログを表示 |

### 登録確認ダイアログ

登録ボタン押下時に確認ダイアログを表示する。

| 項目 | 値 |
|------|-----|
| タイトル | 見積 - 登録・更新 |
| 本文 | 見積情報を登録します。よろしいですか？ |
| 確認ボタン | 登録（青） |
| キャンセルボタン | キャンセル |

### 登録処理フロー

1. 登録ボタンをクリック
2. バリデーション実行（エラー時は処理中断）
3. 登録確認ダイアログを表示
4. 「登録」ボタンクリックで登録API呼び出し
5. 成功時:
   - **必須項目が全て入力されている場合**: 制作見積依頼確認ダイアログを表示
   - **必須項目が未入力の場合**: 登録成功ダイアログを表示（タイトル: 「見積 - 登録・更新」、メッセージ: 「見積を登録しました」）
   - 「閉じる」ボタン押下で登録したデータの登録・更新画面（`/sales/quotes/{quot_id}`）へ遷移
6. 失敗時: `ErrorMessageList`にエラーメッセージを表示

**制作見積依頼確認ダイアログの表示条件（全て入力済み）:**
- 見積件名（quot_subject）
- 品名（prod_name）
- 見積概要（quot_summary）
- 主管センター（center_id）

制作見積依頼確認ダイアログ以降のフローについては、[見積ダイアログ仕様書](./quotes-dialogs.md#制作見積依頼確認ダイアログ新規登録時)を参照。

## 詳細・更新モード

既存の見積データを表示・更新するモード（`/sales/quotes/{id}`）。

### 表示項目（詳細モーダルと統一）

基本情報の表示レイアウトは詳細モーダルと同様。2項目を1行に並べて表示し、値がnullの場合は「-」を表示する。

| 行 | 左項目 | 右項目 |
|----|--------|--------|
| 1 | 【部署】部署コード + 部署名 | 【担当者】担当者の社員名 |
| 2 | 【見積No】見積書番号 | 【得意先】得意先コード + 得意先名 |
| 3 | 【見積件名】見積件名 | 【品名】品名 |
| 4 | 【主管センター】主管センター名 | 【ステータス】ステータスラベル |
| 5 | 【見積金額】見積金額（¥カンマ区切り） | 【見積日】見積日（YYYY/MM/DD形式） |
| 6 | 【提出方法】提出方法ラベル | 【見積結果】見積結果ラベル |
| 7 | 【参考資料】参考資料パス | 【見積書格納先】常に「-」を表示 |

### 見積概要・失注理由（別行表示）

| 項目 | 表示条件 | スタイル |
|------|---------|---------|
| 【見積概要】 | 常時表示 | ラベルの下に枠線付きで表示 |
| 【失注理由】 | 見積結果が失注（20）の場合のみ表示 | 見積概要と同様のスタイル |

### フッターボタン（詳細・更新モード）

ステータスに応じて表示するボタンが変化する。

| quot_status | prod_quot_status | 表示ボタン |
|-------------|------------------|-----------|
| 共通 | - | 一覧に戻る |
| 00（作成中） | 00（制作見積依頼前） | 削除（赤）、更新（青）、制作見積依頼（青） |
| 00（作成中） | 20（制作見積済） | 差戻し（赤）、登録（青） |
| 10（承認待ち） | 30（制作見積受取済） | 登録取消（赤）、更新（青）、承認（青）※1 |
| 20（承認済） | 30（制作見積受取済） | 発行（青）、承認取消（赤）※1 |
| 30（発行済） | 30（制作見積受取済） | 失注登録（青）※2 |

※1 承認・承認取消ボタンはアクセス区分が `00` または `20` のユーザーのみ表示
※2 失注登録は見積結果が未確定（00）の場合のみ表示

### 初期値設定

画面表示時に詳細データから初期値が自動設定される。

- 得意先: `setSelectedCustomer` で得意先オブジェクトをセット（タグ形式で表示）
- 見積件名: 詳細データの見積件名
- 品名: 詳細データの品名
- 見積概要: 詳細データの見積概要
- 提出方法: 詳細データの提出方法
- 参考資料: 詳細データの参考資料パス
- 主管センター: センター名から該当センターを選択
- 伝達事項: 詳細データの伝達事項

### 得意先選択機能

新規登録モードと同様の `CustomerSelectField` コンポーネントを使用。

- **サジェスト**: 入力文字列から得意先を検索（300msデバウンス）
- **得意先選択ダイアログ**: ボタン押下で得意先選択ダイアログを表示
- **選択後の表示**: タグ形式で得意先名を表示、×ボタンでクリア可能

```tsx
<CustomerSelectField
  value={selectedCustomer}
  onChange={handleCustomerChange}
  suggestFn={suggestQuotCustomersForCreate}
  searchFn={searchQuotCustomersForCreate}
  onError={addError}
/>
```

API通信エラーは `ErrorMessageList` で画面上部に表示される。

## 戻る確認ダイアログ（全モード共通）

フォームの値が変更されている状態で「一覧に戻る」ボタンを押した場合、確認ダイアログを表示する。

### 対象となるモード

| モード | 対象フォーム |
|--------|-------------|
| 新規登録モード | 見積登録フォーム（createForm） |
| quot_status=00（作成中） | 見積更新フォーム（createForm） |
| prod_quot_status=20 または quot_status=10（制作見積済/承認待ち） | 見積金額フォーム（quotAmountForm） |
| quot_status=30（発行済） | ステータス60用フォーム（status60Form） |

### 確認ダイアログ

| 項目 | 内容 |
|------|------|
| タイトル | 見積 - 登録・更新 |
| 本文 | 変更した内容は破棄されます。よろしいですか？ |
| キャンセルボタン | キャンセル |
| 確認ボタン | 戻る（青） |

### 動作

- **変更がある場合**: 確認ダイアログを表示し、「戻る」ボタン押下で一覧画面へ遷移
- **変更がない場合**: 確認ダイアログを表示せず、直接一覧画面へ遷移

### 技術的な注意事項

React Hook FormのformState.isDirtyを使用して変更検知を行う。formState.isDirtyはレンダリング時に参照（サブスクリプション）しないと正しく動作しないため、コンポーネントのトップレベルでデストラクチャリングして変数に格納している。

```tsx
const { isDirty: isCreateFormDirty } = createForm.formState;
```
