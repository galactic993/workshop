# 見積ページ仕様書

## 概要

見積の検索・一覧表示を行うページ。

- **URL**: `/sales/quotes`
- **権限**: `sales.quotes.*` または アクセス区分 `00`

## 検索フォーム

### 入力項目

| 項目名 | 入力形式 | 幅 | バリデーション |
|--------|----------|-----|----------------|
| 部署コード | プルダウン | w-96 | 必須（'all' または 6桁以内の半角数字） |
| ステータス | ラジオボタン | - | - |
| 見積No | テキスト | w-96 | 半角数字のみ、11桁以内 |
| 得意先 | テキスト + ボタン | w-96 | - |
| 見積日（開始） | 日付 | w-96（2入力で分割） | - |
| 見積日（終了） | 日付 | w-96（2入力で分割） | 開始日以降 |
| 見積件名 | テキスト | w-96 | 50文字以内 |
| 品名 | テキスト | w-96 | 50文字以内 |

### 部署コードプルダウン

ユーザーのアクセス権限に応じて選択可能な部署コードが決定される。

| アクセス区分 | 選択可能な部署コード | 状態 |
|-------------|---------------------|------|
| 全て（00） | 全部署コード（センター/チームを除く） | 活性 |
| 所長（20） | 所属センター配下 + 参照可能組織配下の部署コード | 活性 |
| 一般（40）参照可能組織あり | 自身の部署コード + 参照可能組織配下の部署コード | 活性 |
| 一般（40）参照可能組織なし | 自身の部署コードのみ | 非活性（1件自動選択） |

選択肢の表示形式: `部署コード 部署名`（例: `262111 東京営業1課`）

「全て」オプション（値: `'all'`）は、活性状態の場合のみ表示される。選択すると全部署を対象に検索する。

選択肢が1件のみの場合、その値が自動選択され、プルダウンは非活性となる。

### バリデーションエラーメッセージ

| 項目 | エラー種別 | メッセージ | 検証場所 |
|------|-----------|-----------|----------|
| 部署コード | 必須 | 部署を選択してください | Zodスキーマ |
| 部署コード | 選択肢外の値 | 有効な値を選択してください | useQuotSearchフック + バックエンド |
| ステータス | 選択肢外の値 | 有効な値を選択してください | Zodスキーマ + バックエンド |
| 見積No | 形式 | 半角数字で入力してください | Zodスキーマ |
| 見積No | 桁数 | 11桁以内で入力してください | Zodスキーマ |
| 見積日（終了） | 範囲 | 開始日は終了日以前の日付を入力してください | Zodスキーマ |
| 見積件名 | 桁数 | 50文字以内で入力してください | Zodスキーマ |
| 品名 | 桁数 | 50文字以内で入力してください | Zodスキーマ |

| エラー種別 | 表示メッセージ |
|-----------|---------------|
| 一覧取得エラー | 見積情報の取得に失敗しました。時間を空けて再度お試しください |
| 部署コード取得エラー | 部署コードの取得に失敗しました。時間を空けて再度お試しください |
| 得意先検索エラー | 得意先の取得に失敗しました。時間を空けて再度お試しください |

### ソート機能

- ソート可能な列ヘッダーをクリックするとソート

## 得意先選択機能

### CustomerSelectField コンポーネント

得意先選択は `CustomerSelectField` コンポーネントで統合管理される。

```tsx
<CustomerSelectField
  value={selectedCustomer}
  onChange={handleCustomerChange}
  suggestFn={suggestQuotCustomers}
  searchFn={searchQuotCustomers}
  onError={addError}
/>
```

API通信エラーは `ErrorMessageList` で画面上部に表示される。

### サジェスト機能

- 得意先入力欄に文字を入力すると、部分一致で候補を表示
- 得意先コードまたは得意先名で検索可能
- 候補選択時に得意先情報をオブジェクトで保持
- **選択後の表示形式**: タグ形式で得意先名を表示、×ボタンでクリア可能
- **デバウンス時間**: 300ms（`useDebouncedValue`フック使用）
- 1文字以上の入力でサジェスト開始

### 得意先選択ダイアログ

- 「得意先選択」ボタンクリックで表示
- 得意先コード・得意先名で検索可能
- 検索結果から得意先を選択し、入力欄に反映
- **Portalを使用**: `document.body`に直接レンダリングされ、親フォームとのネスト問題を回避

#### ダイアログ内バリデーション

| 項目 | ルール | エラーメッセージ |
|------|--------|----------------|
| 得意先コード | 半角数字のみ、5桁以内 | 半角数字で入力してください / 5桁以内で入力してください |
| 得意先名 | 任意、100文字以内 | 100文字以内で入力してください |

### 検索対象

- 自身が参照可能な部署コードのセンターに紐づく部署別得意先から検索
- アクセス区分に応じて検索対象が制限される

### フォームスキーマ

検索フォームでの得意先データは `customer` オブジェクトとして管理される。

```typescript
customer: z
  .object({
    customer_id: z.number().int().positive(),
    customer_cd: z.string(),
    customer_name: z.string(),
  })
  .nullable()
  .optional(),
```

## 承認機能

### 承認権限判定

```typescript
// frontend/src/lib/permissions.ts

export function canApproveQuots(accessType: string | null): boolean {
  return accessType === '00' || accessType === '20';
}
```

### 確認ダイアログ

承認・承認取消ボタン押下時に確認ダイアログを表示する。

#### 承認確認ダイアログ

| 項目 | 値 |
|------|-----|
| タイトル | 見積 |
| 本文 | 選択した見積を承認します。よろしいですか？ |
| 確認ボタン | 承認（青） |
| キャンセルボタン | キャンセル |

#### 承認取消確認ダイアログ

| 項目 | 値 |
|------|-----|
| タイトル | 見積 |
| 本文 | 選択した見積を承認取消します。よろしいですか？ |
| 確認ボタン | 承認取消（赤） |
| キャンセルボタン | キャンセル |

### 処理フロー

1. 承認/承認取消ボタンをクリック
2. 確認ダイアログを表示
3. 「確認」ボタンクリックでAPI呼び出し
4. 成功時: MessageDialog表示、一覧を再取得
5. 失敗時: ErrorMessageListでエラー表示
