# 売上管理ページ仕様

## 概要

売上管理モジュールのページ仕様を定義します。

---

## 売上管理ページ (`/sales`)

**ファイルパス**: `frontend/src/app/sales/page.tsx`

### 概要

売上管理モジュールの第1階層ページ。権限に基づいて第2階層へのリンクカードを表示します。

### アクセス権限

- **アクセス区分「00（すべて）」**: 全てのリンクカードを表示
- **sales.* 権限保持者**: 該当する権限のリンクカードのみ表示
- **権限なし**: ポータル（`/`）へリダイレクト
- **未認証**: ポータル（`/`）へリダイレクト

### 表示カード

| カード | 説明 | 表示条件 |
|--------|------|----------|
| 見積 | 見積書を作成します。 | `sales.quotes.*` |
| 受注 | 受注登録/工番発行を行います。 | `sales.orders.*` |
| 売上 | 社内仕入、売上登録を行います。 | `sales.revenue.*` |
| 月次 | 製品在庫金額一覧、損益集計表などの出力を行います。 | `sales.monthly.*` |

## 部署別得意先メンテナンスページ (`/sales/orders/customer`)

**ファイルパス**: `frontend/src/app/sales/orders/customer/page.tsx`

### 概要

センターごとに部署別得意先の追加・一覧表示を行うメンテナンス画面。

### アクセス権限

- **アクセス区分「00（すべて）」**: アクセス可能
- **sales.orders.customer 権限保持者**: アクセス可能
- **権限なし**: ポータル（`/`）へリダイレクト
- **未認証**: ポータル（`/`）へリダイレクト

### 機能

1. **センター選択**
   - アクセス可能なセンターをプルダウンで選択
   - 選択肢が1つの場合は自動選択・非活性

2. **得意先入力**
   - テキスト入力でサジェスト検索（得意先コード/得意先名）
   - スペース区切りでAND検索
   - 既にセンターに紐づいている得意先は候補から除外
   - サジェストから選択して得意先を指定
   - デバウンス処理（300ms）で過剰なAPI呼び出しを防止

3. **得意先選択ボタン**
   - モーダルダイアログで得意先を検索・選択
   - 得意先コード/得意先名で個別に検索可能
   - 既にセンターに紐づいている得意先は候補から除外
   - ラジオボタン形式で選択、選択ボタンで入力欄に反映

4. **得意先選択フィールド（CustomerSelectField）**
   - サジェスト入力とダイアログ選択を統合したコンポーネント
   - 選択後はタグ形式で得意先名を表示
   - ×ボタンで選択をクリア可能
   - 詳細は[フォームコンポーネント](../components/forms.md)を参照

5. **追加ボタン**
   - 選択した得意先をセンターの部署コードに紐づけ
   - フォームバリデーション実行後、確認ダイアログを表示
   - 得意先未選択時は入力欄の下にエラーメッセージを表示
   - 成功時: MessageDialog「登録しました」、フォームクリア、一覧再取得
   - 失敗時: MessageDialogでエラーメッセージ表示

5. **一覧表示**
   - センターに紐づいている部署別得意先を表示
   - 表示件数切り替え（10/25/50/100件）
   - ページネーション対応（ページ番号ボタン形式）
   - 得意先コードでソート可能（昇順/降順トグル）

6. **削除機能**
   - 一覧の各行に削除ボタン
   - 確認ダイアログ表示後にAPI実行
   - 成功時: MessageDialog「削除しました」、一覧再取得
   - 失敗時: MessageDialogでエラーメッセージ表示

### バリデーション

**スキーマファイル**: `frontend/src/schemas/sectionCustomerSchema.ts`

| フィールド | ルール | エラーメッセージ |
|-----------|--------|-----------------|
| center_id | 必須（文字列、1文字以上） | センターを選択してください |
| customer | 必須（得意先オブジェクト） | 得意先を選択してください |

**得意先オブジェクトの構造**:
```typescript
{
  customer_id: number;    // 得意先ID（正の整数）
  customer_cd: string;    // 得意先コード
  customer_name: string;  // 得意先名
}
```

**型定義**:
- `SectionCustomerFormInput`: フォーム入力型（customerがnullable）
- `SectionCustomerFormData`: バリデーション後の出力型

### 実装の特記事項

- センター変更時のフォーム自動リセット機能（得意先選択をクリア）
- `CustomerSelectField`コンポーネントを使用し、サジェストとダイアログ選択を統合
- フォームバリデーションは`trigger('customer')`で実行し、エラーは入力欄の下に表示
