# 受注管理ページ仕様

## 概要

受注管理モジュールのページ仕様を定義します。

---

## 受注情報取込ページ (`/sales/orders/import`)

**ファイルパス**: `frontend/src/app/sales/orders/import/page.tsx`

### 概要

Excelファイルをアップロードして受注情報をバリデーションするページ。バリデーション結果を画面に表示し、エラー時はエラーログをテキストエリアに表示します。

### アクセス権限

- **アクセス区分「00（すべて）」**: アクセス可能
- **sales.orders.import 権限保持者**: アクセス可能
- **権限なし**: ポータル（`/`）へリダイレクト
- **未認証**: ポータル（`/`）へリダイレクト

### 機能

1. **ファイル選択**
   - ファイル選択ボタンをクリックするとファイルピッカーが開く
   - Excelファイル（.xlsx）のみ選択可能
   - 選択済みのファイル名とファイルサイズを表示
   - ファイルサイズの自動フォーマット（B、KB、MB単位）

2. **取込ボタン**
   - ファイル選択後に取込ボタンが活性化
   - 取込ボタンをクリックすると確認ダイアログが表示
   - 取込中はボタンが無効化され「取込中...」と表示
   - API呼び出し時にFormDataを使用してファイル送信

3. **バリデーション**
   - フロントエンド: ファイル形式（.xlsx）、ファイルサイズ（1MB以下）の確認
   - バックエンド: Excelデータの詳細なバリデーションを実行
   - バリデーションエラー時はエラーメッセージを表示

4. **エラーログ表示**
   - バリデーションエラーが発生した場合、エラーログをテキストエリアに表示
   - テキストエリアは読み取り専用
   - 複数のエラーは改行で区切られて表示
   - エラーの解析: `errors` 配列内の1行ごとにエラーメッセージを含む

5. **エラーログコピー機能**
   - エラーログテキストエリアの上に「コピー」ボタンを配置
   - クリックするとエラーログ全体をクリップボードにコピー
   - クリップボードコピーはnavigator.clipboard APIを使用
   - コピー成功時は特に確認メッセージを表示しない（一般的なWebアプリケーション設計）

### スタイル仕様

**メインカード**:
- 背景色: 白 (`bg-white`)
- ボーダー: `border border-gray-200`
- 角丸: `rounded-lg`
- パディング: 24px (`p-6`)
- シャドウ: `shadow-sm`

**取込データセクション**:
- フレックスレイアウト（行方向）
- ラベル幅: 固定 (`w-40`)
- ファイル情報とボタンは同一行に配置
- 取込ボタンは右端に配置（`ml-auto`）

**ファイル選択ボタン**:
- 背景色: 薄青 (`bg-blue-50`)
- テキスト色: 青 (`text-blue-700`)
- パディング: 8px 16px (`px-4 py-2`)
- ホバー時: `bg-blue-100`
- フォーカス: `ring-2 ring-blue-500 ring-offset-2`

**取込ボタン**:
- 背景色: 濃青 (`bg-blue-600`)
- テキスト色: 白 (`text-white`)
- パディング: 8px 16px (`px-4 py-2`)
- シャドウ: `shadow-sm`
- ホバー時: `bg-blue-700`
- フォーカス: `ring-2 ring-blue-500 ring-offset-2`
- 無効時: 50%透明 + カーソル変更

**テキストエリア**:
- 高さ: 128px (`h-32`)
- ボーダー: `border border-gray-300`
- 背景色: 薄灰 (`bg-gray-50`)
- テキスト色: 灰 (`text-gray-700`)
- パディング: 8px 12px (`px-3 py-2`)
- リサイズ不可: `resize-none`
- 角丸: `rounded-md`

**コピーボタン**:
- 背景色: 薄灰 (`bg-gray-100`)
- テキスト色: 灰 (`text-gray-700`)
- パディング: 6px 12px (`px-3 py-1.5`)
- ホバー時: `bg-gray-200`
- フォーカス: `ring-2 ring-gray-500 ring-offset-2`

**エラーメッセージ**:
- テキスト色: 赤 (`text-red-600`)
- フォントサイズ: 小 (`text-sm`)

### バリデーション

**スキーマファイル**: `frontend/src/schemas/orderImportSchema.ts`

| フィールド | ルール | エラーメッセージ |
|-----------|--------|-----------------|
| file | 必須（FileList） | 取込データを選択してください |
| file | Excel形式（.xlsx） | Excel形式（.xlsx）のファイルを選択してください |
| file | ファイルサイズ≤1MB | ファイルサイズは1MB以下にしてください |

### ダイアログ

#### 確認ダイアログ

- **タイトル**: 「受注情報取込」
- **メッセージ**: 「受注情報を取り込んで工番取得処理を実行します。\nよろしいですか？」
- **ボタン**: 「取込」「キャンセル」
- **取込中表示**: ボタンは無効化され「取込中...」と表示

#### メッセージダイアログ

バリデーション結果を表示するメッセージダイアログ。

**成功時**:
- **タイトル**: 「受注情報取込」
- **メッセージ**: 「{first_order_cd}から{n}件の工番を発行しました。\n受注登録画面で必ず受注内容の更新をしてください。」

**エラー時**:
- **タイトル**: 「受注情報取込」
- **メッセージ**: 「受注情報の取り込みに失敗しました。\nエラーログを確認してください。」

**その他エラー**:
- **タイトル**: 「受注情報取込」
- **メッセージ**: 「受注情報の取込に失敗しました。\n時間を空けて再度お試しください。」

### 実装の特記事項

- React Hook Form を使用したフォーム管理
- Zod スキーマによるバリデーション
- FormData を使用したマルチパートファイル送信
- API呼び出しには `api` モジュール（axios）を使用
- ファイルサイズはバイト単位で自動的に適切な単位（B、KB、MB）にフォーマット
- エラーログはAPI レスポンスの `errors` 配列をそのまま表示
