# 要件定義

## 機能要件（第1フェーズ：MVP）

### 1. 認証・認可機能

#### 1.1 ログイン ✅ 実装完了
- 部署コード + 社員コードによるパスワードレス認証
- Laravel Sanctum SPA認証（セッションベース）
- Redisセッションストレージ
- CSRF保護

#### 1.2 権限管理 ✅ 実装完了
- 役割ベースアクセス制御（RBAC）
- アクセス区分による権限制御（00:全て, 10:ディレクター, 20:所長, 30:リーダー, 40:一般）
- 機能ごとの権限設定（permissions テーブル）
- 役割ごとの権限割当（permission_role テーブル）
- 社員への役割割当（employee_role テーブル）
- 参照可能組織（visible_departments テーブル）

#### 1.3 得意先関連マスタ ✅ 実装完了
- 業種マスタ（industries テーブル）
- 得意先グループマスタ（customer_groups テーブル）
- 会社団体マスタ（companies テーブル）
- 得意先マスタ（customers テーブル）
- 部署別得意先（customer_section_cd テーブル）

#### 1.4 ログ管理（実装予定）
- アクセスログの記録
- 操作ログの記録
- ログの閲覧・検索

## 非機能要件

### 1. パフォーマンス

| 項目 | 要件 |
|------|------|
| ページ読み込み時間 | 2秒以内 |
| API レスポンスタイム | 500ms以内（通常時） |
| 同時接続ユーザー数 | 30名 |
| データ件数（初期） | 10万件程度 |

### 2. セキュリティ

#### 2.1 通信
- HTTPS による暗号化通信
- TLS 1.2 以上

#### 2.2 認証
- セッションベース認証（Laravel Sanctum SPA認証）
- Redisセッションストレージ
- CSRF保護

#### 2.3 データ保護
- SQLインジェクション対策
- XSS対策
- CSRF対策

#### 2.4 アクセス制御
- 役割ベースアクセス制御（RBAC）
- IP制限（将来対応）
- 二要素認証（将来対応）

#### 2.5 監査
- アクセスログの記録
- 操作ログの記録
- ログの改ざん防止

### 3. 可用性

| 項目 | 要件 |
|------|------|
| 稼働時間 | 平日 8:00〜20:00 |
| 目標稼働率 | 99%以上 |
| バックアップ | 日次自動バックアップ |
| 復旧時間目標（RTO） | 4時間以内 |

### 4. 保守性

- コードの可読性・保守性を重視
- ドキュメントの整備
- テストコードの作成
- ログの適切な出力

### 5. 拡張性

- 機能追加に対応できる設計
- ユーザー数増加に対応可能
- データ量増加に対応可能

### 6. ユーザビリティ

- 直感的な操作性
- エラーメッセージの分かりやすさ
- ヘルプ・マニュアルの提供

### 7. 互換性

#### 7.1 ブラウザ対応
- Google Chrome（最新版）
- Microsoft Edge（最新版）

#### 7.2 デバイス対応
- PC（Windows, Mac）

### 8. データ要件

#### 8.1 データ保持期間
- 業務データ: 5年以上
- ログデータ: 1年以上

#### 8.2 データバックアップ
- 日次自動バックアップ
- バックアップデータの保持期間: 30日

#### 8.3 データ整合性
- トランザクション処理の徹底
- データ整合性チェック機能

## 制約事項

1. **技術スタック**
   - フロントエンド: Next.js
   - バックエンド: Laravel
   - データベース: PostgreSQL

2. **開発環境**
   - 社内ネットワーク環境での開発
   - Git によるバージョン管理

3. **運用環境**
   - 社内サーバーでの運用
   - HTTPS必須
