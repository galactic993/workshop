# ユーティリティ・フックテスト仕様

## 1. permissions.test.ts

**ファイル**: `frontend/src/lib/permissions.test.ts`

権限判定関数のテスト。

### hasPermission

| テストケース | チェック内容 |
|-------------|-------------|
| アクセス区分00は常にtrue | `access_type='00'` の場合、権限配列に関わらず常に `true` を返す |
| プレフィックス（ワイルドカード）で判定できる | `'sales.'` のようなプレフィックスで `sales.orders.customer` などにマッチする |
| 完全一致で判定できる | `'sales.orders.customer'` の完全一致で判定できる |
| 権限がない場合はfalse | 権限配列が空の場合、一般ユーザーは `false` を返す |
| accessTypeがnullの場合は権限ベースで判定 | `access_type=null` の場合は権限配列のみで判定する |

### canAccessSales

| テストケース | チェック内容 |
|-------------|-------------|
| アクセス区分00はアクセス可能 | 管理者は常にアクセス可能 |
| sales.*権限があればアクセス可能 | `sales.` で始まる権限があればアクセス可能 |
| sales権限がなければアクセス不可 | sales権限がない場合はアクセス不可 |

### canAccessSectionCustomer

| テストケース | チェック内容 |
|-------------|-------------|
| アクセス区分00はアクセス可能 | 管理者は常にアクセス可能 |
| sales.orders.customer権限があればアクセス可能 | 部署別得意先管理権限があればアクセス可能 |
| 異なる権限ではアクセス不可 | `sales.quotes.create` などでは部署別得意先にアクセス不可 |

### canAccessQuots

| テストケース | チェック内容 |
|-------------|-------------|
| アクセス区分00はアクセス可能 | 管理者は常にアクセス可能 |
| sales.quotes.*権限があればアクセス可能 | 見積関連権限（create/approve/issue）があればアクセス可能 |
| 異なる権限ではアクセス不可 | `sales.orders.customer` などでは見積にアクセス不可 |

## 4. useDebounce.test.ts

**ファイル**: `frontend/src/hooks/useDebounce.test.ts`

デバウンスフックのテスト。FakeTimersを使用。

### useDebounce

| テストケース | チェック内容 |
|-------------|-------------|
| デバウンスされたコールバックが遅延後に実行される | 300ms後にコールバックが1回だけ実行される |
| 連続して呼び出すと最後の呼び出しのみ実行される | 連続呼び出し時、最後の値でのみコールバックが実行される |

### useDebouncedValue

| テストケース | チェック内容 |
|-------------|-------------|
| 初期値を返す | 初期レンダリング時に初期値が返される |
| 遅延後に新しい値を返す | 値変更後、300ms経過すると新しい値が反映される |
| 連続して値が変わると最後の値になる | 連続変更時、最終的に最後の値になる |
