# UIコンポーネントテスト仕様

## 1. Pagination.test.tsx

**ファイル**: `frontend/src/components/ui/Pagination.test.tsx`

ページネーションコンポーネントのテスト。

### 表示/非表示

| テストケース | チェック内容 |
|-------------|-------------|
| totalPages <= 1 の場合は何も表示しない | `totalPages=1` で `container.firstChild` が `null` |
| totalPages = 0 の場合は何も表示しない | `totalPages=0` で何も表示されない |
| totalPages > 1 の場合はページネーションを表示する | `totalPages=2` でページ番号が表示される |

### 少ないページ数の場合

| テストケース | チェック内容 |
|-------------|-------------|
| totalPages が maxVisiblePages + 2 以下なら全ページを表示 | 7ページで1〜7全て表示、省略記号なし |

### 多いページ数の場合

| テストケース | チェック内容 |
|-------------|-------------|
| 先頭付近（page 1-3）では末尾に省略記号が表示される | `1,2,3,4,...,10` 形式 |
| 末尾付近では先頭に省略記号が表示される | `1,...,7,8,9,10` 形式 |
| 中間ページでは両側に省略記号が表示される | `1,...,4,5,6,...,10` 形式、`...` が2つ |

### 前へボタン

| テストケース | チェック内容 |
|-------------|-------------|
| page 1 では無効化されている | 最初のページで `<` ボタンが `disabled` |
| page 2 以降では有効 | 2ページ目以降で `<` ボタンが有効 |
| クリックで onPageChange(currentPage - 1) が呼ばれる | クリックで前ページへのコールバック |

### 次へボタン

| テストケース | チェック内容 |
|-------------|-------------|
| 最終ページでは無効化されている | 最終ページで `>` ボタンが `disabled` |
| 最終ページ以外では有効 | 途中ページで `>` ボタンが有効 |
| クリックで onPageChange(currentPage + 1) が呼ばれる | クリックで次ページへのコールバック |

### ページ番号クリック

| テストケース | チェック内容 |
|-------------|-------------|
| クリックで onPageChange がそのページ番号で呼ばれる | ページ3クリックで `onPageChange(3)` |
| 現在のページ番号もクリック可能 | 現在ページクリックでもコールバック発火 |

### 現在のページのスタイル

| テストケース | チェック内容 |
|-------------|-------------|
| 現在のページには青いスタイルが適用される | `bg-blue-500`, `text-white` クラス |
| 他のページには白いスタイルが適用される | `bg-white`, `text-gray-700` クラス |

## 2. ErrorBoundary.test.tsx

**ファイル**: `frontend/src/components/ui/ErrorBoundary.test.tsx`

エラー境界コンポーネントのテスト。

### 正常時

| テストケース | チェック内容 |
|-------------|-------------|
| 子コンポーネントを正常に表示する | 子コンテンツがDOMに存在する |
| エラーがない場合、フォールバックは表示されない | フォールバックがDOMに存在しない |

### エラー発生時

| テストケース | チェック内容 |
|-------------|-------------|
| エラー発生時にデフォルトのフォールバックUIを表示する | 「エラーが発生しました」とエラーメッセージを表示 |
| カスタムフォールバックが指定されている場合はそれを表示する | `fallback` propのUIが表示される |
| onErrorコールバックが呼ばれる | `onError(error, errorInfo)` が呼ばれ、componentStackを含む |

### 再試行ボタン

| テストケース | チェック内容 |
|-------------|-------------|
| デフォルトで再試行ボタンが表示される | 「再試行」ボタンがDOMに存在する |
| canRetry=false の場合、再試行ボタンは表示されない | ボタンが非表示 |
| 再試行ボタンクリックでエラー状態がリセットされる | クリック後に正常コンテンツが表示される |

### エラーメッセージ表示

| テストケース | チェック内容 |
|-------------|-------------|
| エラーメッセージがない場合はデフォルトメッセージを表示 | 「予期せぬエラーが発生しました」 |
| エラーメッセージがある場合はそれを表示 | エラーオブジェクトのmessageを表示 |

### console.error

| テストケース | チェック内容 |
|-------------|-------------|
| エラー発生時にconsole.errorが呼ばれる | `console.error` がモック検証で呼ばれている |

## 3. CustomerSelectDialog.test.tsx

**ファイル**: `frontend/src/components/forms/CustomerSelectDialog.test.tsx`

得意先選択ダイアログコンポーネントのテスト。

### 表示/非表示

| テストケース | チェック内容 |
|-------------|-------------|
| isOpen=false の場合は何も表示しない | `container.firstChild` が `null` |
| isOpen=true の場合はダイアログを表示する | 「得意先選択」タイトルがDOMに存在する |
| カスタムタイトルを表示できる | `title="商品選択"` で「商品選択」が表示される |

### フィールドラベル

| テストケース | チェック内容 |
|-------------|-------------|
| デフォルトのラベルが表示される | 「得意先コード」「得意先名」がDOMに存在する |
| カスタムラベルを表示できる | `fieldLabels` propで「商品コード」「商品名」が表示される |

### バリデーション

| テストケース | チェック内容 |
|-------------|-------------|
| 得意先コードに半角数字以外を入力して検索するとエラー | 「半角数字で入力してください」エラー表示、検索関数は呼ばれない |
| 得意先コードに5桁超入力すると警告 | `maxLength=5` で制限、5桁までしか入力できない |

### 検索機能

| テストケース | チェック内容 |
|-------------|-------------|
| 検索ボタンクリックで searchFn が呼ばれる | 検索ボタンクリックでコールバック発火 |
| 検索結果が表示される | 得意先コード・名前がDOMに存在する |
| 検索結果が0件の場合「該当する得意先がありません」を表示 | 空結果時のメッセージ表示 |
| 検索失敗時に onError が呼ばれる | `success: false` 時に `onError(message)` が呼ばれる |

### 選択機能

| テストケース | チェック内容 |
|-------------|-------------|
| ラジオボタンで得意先を選択できる | ラジオボタンクリックで `checked` 状態になる |
| 選択ボタンは得意先未選択時に disabled | 未選択時は選択ボタンが `disabled` |
| 選択確定で onSelect が呼ばれダイアログが閉じる | 選択ボタンクリックで `onSelect(customer)` と `onClose()` が呼ばれる |

### キャンセル・閉じる

| テストケース | チェック内容 |
|-------------|-------------|
| キャンセルボタンで onClose が呼ばれる | キャンセルボタンクリックで `onClose()` が呼ばれる |
| オーバーレイクリックで onClose が呼ばれる | 背景クリックで `onClose()` が呼ばれる |

### codeMaxLength

| テストケース | チェック内容 |
|-------------|-------------|
| デフォルトで5文字まで入力可能 | `maxLength="5"` 属性 |
| カスタムの最大文字数を設定できる | `codeMaxLength={10}` で `maxLength="10"` |
