# 見積APIテスト仕様

**ファイル**: `backend/tests/Feature/QuotApiTest.php`

## 概要

見積API（CRUD、承認、発行）のテスト。
認証・権限チェック、バリデーション、ビジネスロジックを検証。

## エンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/api/quotes` | 見積一覧 |
| GET | `/api/quotes/access-info` | アクセス情報 |
| POST | `/api/quotes` | 見積新規登録 |
| POST | `/api/quotes/{id}/approve` | 承認 |
| POST | `/api/quotes/{id}/cancel-approve` | 承認取消 |
| POST | `/api/quotes/{id}/request-production` | 制作見積依頼 |
| POST | `/api/quotes/{id}/reject` | 差戻し |
| GET | `/api/quotes/customers/suggest` | 得意先サジェスト |
| GET | `/api/quotes/customers/search` | 得意先検索 |
| GET | `/api/quotes/customers/suggest-for-create` | 新規登録用サジェスト |
| GET | `/api/quotes/customers/search-for-create` | 新規登録用検索 |

## GET /api/quotes

### 認証・権限チェック

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| 未認証ユーザーは見積一覧を取得できない | 「ログインしてください」 | 401 |
| 権限のないユーザーは見積一覧を取得できない | 「アクセス権限がありません」 | 403 |
| アクセス区分00（全て）のユーザーは取得できる | 管理者はアクセス可能 | 200 |
| sales.quotes.create権限を持つユーザーは取得できる | 見積権限ありでアクセス可能 | 200 |

### ページネーション・検索

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| ページネーションパラメータが正しく動作する | `page=1&per_page=25` | 200 |
| 検索パラメータが正しく動作する | `quote_no=2024-001&status=00` | 200 |
| 見積件名検索が正しく動作する | `quot_subject=テスト` | 200 |
| ソートパラメータが正しく動作する | `sort_field=quote_no&sort_order=asc` | 200 |

**レスポンス構造**:
```json
{
  "success": true,
  "quotes": [...],
  "total": 100,
  "page": 1,
  "per_page": 10,
  "total_pages": 10,
  "access_info": {
    "default_section_cd": "262000",
    "is_section_cd_disabled": false
  }
}
```

## POST /api/quotes（新規登録）

### 認証・権限チェック

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| 未認証ユーザーは見積を登録できない | 「ログインしてください」 | 401 |
| 権限のないユーザーは見積を登録できない | 「アクセス権限がありません」 | 403 |

### バリデーション

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| バリデーションエラー（品名なし） | prod_name空でエラー | 422 |
| バリデーションエラー（得意先なし） | customer_id未指定でエラー | 422 |

### 正常系

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| 正常に見積を登録できる | 必須項目入力で登録成功 | 200 |
| 見積登録時にステータスが00（登録済）になる | ステータス初期値確認 | 200 |

**レスポンス**:
```json
{
  "success": true,
  "quot_id": 123,
  "quot_number": "26-00001",
  "message": "登録しました"
}
```

## POST /api/quotes/{id}/approve（承認）

### 認証・権限チェック

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| 未認証ユーザーは見積を承認できない | 「ログインしてください」 | 401 |
| 権限のないユーザーは見積を承認できない | 一般ユーザー（40）は承認不可 | 403 |
| アクセス区分00（全て）のユーザーは承認できる | 管理者は承認可能 | 200 |
| アクセス区分20（所長）のユーザーは承認できる | 所長は承認可能 | 200 |

### エラーケース

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| 存在しない見積は承認できない | 「見積が見つかりません」 | 404 |
| 承認待ち以外のステータスは承認できない | 「承認待ちの見積のみ承認できます」 | 400 |

### 正常系

| テストケース | チェック内容 |
|-------------|-------------|
| 承認後のステータス | quot_status = '50'（承認済） |
| 承認者情報 | approved_by = employee_id |
| 承認日時 | approved_at が設定される |

## 新規登録用得意先API

### GET /api/quotes/customers/suggest-for-create

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| 未認証ユーザーはアクセスできない | 「ログインしてください」 | 401 |
| 権限のないユーザーはアクセスできない | 「アクセス権限がありません」 | 403 |
| 正常にサジェストが動作する | query指定で検索 | 200 |
| スペース区切りでAND検索できる | `query=東京 商事` | 200 |
| 最大20件まで返す | 結果が20件以下 | 200 |

| 両方のパラメータで検索できる | `customer_cd=001&customer_name=商事` | 200 |
