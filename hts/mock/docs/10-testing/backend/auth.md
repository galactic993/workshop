# 認証APIテスト仕様

**ファイル**: `backend/tests/Feature/AuthApiTest.php`

## 概要

認証API（ログイン、ログアウト、認証状態確認）のテスト。
レート制限の検証も含む。

## エンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| POST | `/api/login` | ログイン |
| POST | `/api/logout` | ログアウト |
| GET | `/api/auth/check` | 認証状態確認 |

---

## POST /api/login

### ログイン成功

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| 正しい認証情報でログインできる | 部署コード + 社員コードで認証成功 | 200 |

**レスポンス構造**:
```json
{
  "success": true,
  "user": {
    "employee_id": 1,
    "employee_cd": "000001",
    "employee_name": "管理太郎",
    "section_cd": "000000",
    "section_name": "本部",
    "access_type": "00",
    "roles": [],
    "permissions": []
  }
}
```

### ログイン失敗

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| 不正な認証情報でログインできない | 存在しない部署/社員コードでエラー | 401 |

**レスポンス**:
```json
{
  "success": false,
  "message": "部署コードまたは社員コードが正しくありません"
}
```

### バリデーション

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| バリデーションエラーが返される | 空の部署コード・社員コードでエラー | 422 |

**レスポンス構造**:
```json
{
  "success": false,
  "errors": {
    "section_cd": ["..."],
    "employee_cd": ["..."]
  }
}
```

### レート制限

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| IPアドレス単位のレート制限が機能する（10回/分） | 同一IPから11回目でブロック | 429 |
| アカウント単位のレート制限が機能する（5回/分） | 同一アカウントへ6回目でブロック | 429 |
| 異なるアカウントは別々にカウントされる | アカウントAで5回失敗後、アカウントBは1回目として処理 | 401 |

**レート制限レスポンス**:
```json
{
  "success": false,
  "message": "ログイン試行回数が多すぎます。しばらく待ってから再度お試しください。"
}
```

または

```json
{
  "success": false,
  "message": "このアカウントへのログイン試行回数が多すぎます。しばらく待ってから再度お試しください。"
}
```

## GET /api/auth/check

### 認証状態確認

| テストケース | チェック内容 | HTTPステータス |
|-------------|-------------|---------------|
| 認証状態を確認できる（未認証） | セッションなしで `authenticated: false` | 200 |
| 認証状態を確認できる（認証済み） | セッションありで `authenticated: true` と `user` 情報 | 200 |

**未認証レスポンス**:
```json
{
  "authenticated": false
}
```

**認証済みレスポンス**:
```json
{
  "authenticated": true,
  "user": {
    "employee_id": 1,
    "employee_cd": "000001",
    "employee_name": "テストユーザー",
    "section_cd": "000000"
  }
}
```
