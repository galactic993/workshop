# 受注情報取込API

## 概要

ExcelファイルをアップロードしてExcelファイルのバリデーションを行うAPIを定義します。

**コントローラー**: `backend/app/Http/Controllers/Api/OrderImportController.php`

**サービス**: `backend/app/Services/OrderImportService.php`

---

## 受注情報取込バリデーション・登録

Excelファイルのデータをバリデーションし、成功時は自動的に登録処理を実行します。

### リクエスト

```
POST /api/orders/import
```

### リクエストボディ

```
Content-Type: multipart/form-data

file: Excel ファイル（.xlsx のみ対応）
```

### レスポンス（成功時）

```json
{
  "success": true,
  "data": {
    "first_order_cd": "26-00126",
    "row_count": 5
  },
  "message": "受注情報を登録しました。"
}
```

### レスポンス（バリデーションエラー時）

```json
{
  "success": false,
  "errors": [
    "受注情報リストの件数が多すぎます。\n※30件まで工番の採番・発行をすることができます。",
    "複数の得意先を指定することはできません。",
    "品名は必須です(3行目)"
  ]
}
```

### エラーレスポンス

**ファイル形式エラー（400）**:
```json
{
  "success": false,
  "errors": [
    "Excelファイル（.xlsx, .xls）を選択してください。"
  ]
}
```

**ファイルサイズエラー（400）**:
```json
{
  "success": false,
  "errors": [
    "ファイルサイズは10MB以下にしてください。"
  ]
}
```

**Excel読み込みエラー（400）**:
```json
{
  "success": false,
  "errors": [
    "Excelファイルの読み込みに失敗しました。ファイル形式を確認してください。"
  ]
}
```

**予期しないエラー（500）**:
```json
{
  "success": false,
  "errors": [
    "予期しないエラーが発生しました。"
  ]
}
```

## Excelファイル形式

### 必須列

| 列 | フィールド | 型 | 説明 |
|-----|-----------|-----|------|
| A | 発注書番号 | 文字列 | 発注書番号（最大20桁） |
| B | 得意先コード | 文字列 | 得意先コード |
| C | 品名（略称） | 文字列 | 品名（最大50字） |
| D | 受注日 | 日付 | 受注日（YYYY/MM/DD形式） |

### サンプル

```
発注書番号          得意先コード  品名(略称)         受注日
PO-20260129-001     00001        製品A             2026/01/29
PO-20260129-002     00001        製品B             2026/01/29
PO-20260129-003     00001        製品C             2026/01/30
```

## バリデーション仕様

バリデーションは以下の順序で実施されます。複数のエラーが発生する場合は、すべてのエラーが `errors` 配列に含められます。

### グローバルバリデーション

これらのチェックは件数に関わることや複数行の比較を行い、1つまたは複数の行に影響するエラーです。

#### チェック1: 件数が30件より多い

受注情報リストの行数が30件を超える場合、エラーを返却します。

**エラーメッセージ**:
```
受注情報リストの件数が多すぎます。
※30件まで工番の採番・発行をすることができます。
```

#### チェック2: 件数が1件のみ

受注情報リストの行数が1件のみの場合、エラーを返却します。

**エラーメッセージ**:
```
1件だけの工番の採番・発行をすることはできません。
受注登録を行い、工番の発行をしてください。
```

#### チェック3: 得意先コードが複数存在

リスト内に複数の異なる得意先コードが含まれている場合、エラーを返却します。

**エラーメッセージ**:
```
複数の得意先を指定することはできません。
```

#### チェック4: 得意先コードが得意先マスタに存在しない

リストの得意先コードが `customers` テーブルに存在しない場合、エラーを返却します。

**エラーメッセージ**:
```
リストの得意先コードは得意先マスタに登録されていません。
```

#### チェック5: 得意先コードが部署別得意先に登録されていない

リストの得意先が、ログインユーザーのセンターの部署別得意先（`customer_section_cd` テーブル）に登録されていない場合、エラーを返却します。

**ロジック**:
1. ログインユーザーの所属部署を取得
2. 所属センターを特定（センターフラグが立っている場合はそれ自体、チームの場合は所属するセンターから取得）
3. センターの組織別部署コード（`department_section_cd`）を取得
4. その部署コードに紐づく得意先かを確認

**エラーメッセージ**:
```
リストの得意先コードは部署別得意先に登録されていません。
```

### 行ごとバリデーション

これらのチェックは各行ごとに実施され、エラーメッセージに行番号が含まれます。

#### チェック6: 発注書番号が20桁超

列Aの発注書番号が20文字を超える場合、エラーを返却します。

**エラーメッセージ**:
```
発注書番号は20桁以内で入力してください(n行目)
```

#### チェック7: 品名が空

列Cの品名が空の場合、エラーを返却します。

**エラーメッセージ**:
```
品名は必須です(n行目)
```

#### チェック8: 品名が50桁超

列Cの品名が50文字を超える場合、エラーを返却します。

**エラーメッセージ**:
```
品名は50桁以内で入力してください(n行目)
```

#### チェック9: 受注日が空

列Dの受注日が空の場合、エラーを返却します。

**エラーメッセージ**:
```
受注日は必須です(n行目)
```

#### チェック10: 受注日が不正な日付

列Dの受注日が有効な日付でない場合、エラーを返却します。以下の形式をサポート:
- Excelシリアル値（数値）
- YYYY/MM/DD 形式
- YYYY-MM-DD 形式

**エラーメッセージ**:
```
有効な日付を入力してください(n行目)
```

## 日付パース仕様

受注日は以下の形式でサポートされます。

### 形式1: Excelシリアル値

Excelの標準シリアル値から日付に変換します。

**例**:
```
45,337 → 2026-01-29
```

### 形式2: YYYY/MM/DD

年月日をスラッシュで区切った形式。

**例**:
```
2026/01/29 → 2026-01-29
2026/1/9 → 2026-01-09
```

### 形式3: YYYY-MM-DD

年月日をハイフンで区切った形式。

**例**:
```
2026-01-29 → 2026-01-29
2026-1-9 → 2026-01-09
```

## 空行の扱い

行の4つのカラム（A～D）がすべて空の場合、その行はスキップされます。

## 登録処理仕様

#### 登録対象テーブル

1. **orders テーブル（受注テーブル）**
   - order_cd_year: 受注年号（西暦下2桁）
   - order_cd: 工番通番（採番値）
   - purchase_order_number: 発注書番号
   - customer_id: 得意先ID
   - section_cd_id: 受注部署ID（ログインユーザーの部署から取得）
   - employee_id: 担当者ID（ログインユーザーの社員IDから取得）
   - center_section_cd_id: 主管センターID（ログインユーザーの所属センターから取得）
   - person_in_charge_id: 主管担当者ID（ログインユーザーの社員IDから取得）
   - short_prod_name: 品名（略称）
   - prod_name: 品名（短縮形として品名を使用）
   - official_prod_name: 正式品名（品名と同じ値を使用）
   - quantity: 数量（1で固定）
   - order_on: 受注日
   - order_status: ステータス（00：工番発行済で固定）
   - その他カラムは NULL または デフォルト値を使用

2. **order_operation_amounts テーブル（作業部門別受注金額テーブル）**
   - order_id: 登録した受注レコードのID
   - operation_id: 作業部門ID（全作業部門を登録）
   - 金額: 0で初期化

3. **work_number_relations テーブル（工番関連テーブル）**
   - 複数工番の場合、最初の工番を代表工番として関連情報を記録
   - representative_order_id: 代表工番の受注ID
   - related_order_id: 関連工番の受注ID

#### トランザクション処理

- 全ての登録を1つのトランザクション内で実行
- エラー発生時は全件ロールバック
- テーブルロック: 工番採番時に排他ロックを使用

#### 工番採番処理

```
工番形式: YYCCCCC
YY: 受注年号（西暦下2桁、例: 26）
CCCCC: 工番通番（5桁、001から開始）

採番手順:
1. work_numbers テーブルから年号ごとの最大通番を排他ロック付きで取得
2. 最大通番 + 1 を新通番として採番
3. 採番結果を work_numbers テーブルに記録（INSERT または UPDATE）
```
