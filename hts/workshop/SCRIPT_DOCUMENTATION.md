# Excel クロスチェック スクリプト説明書

## 概要

`check_excel_cross_reference.py` は、2つのExcelファイルを比較して、画面設計書で定義されたフィールドがテーブル定義書に正しく存在するかを検証するPythonスクリプトです。

## 対象ファイル

### 1. 画面設計書（Screen Design Document）
- **ファイル**: `02-02_2_編-制作見積-制作見積書作成トップ_画面設計書_1007.xlsx`
- **シート**: `DB登録値`
- **目的**: DB登録時に使用するフィールド定義

### 2. テーブル定義書（Table Definition Document）
- **ファイル**: `編-テーブル定義書_1012.xlsx`
- **シート**: 各テーブル名（表紙、変更履歴、テーブル一覧 を除く17個のテーブル）
- **目的**: データベーステーブルとフィールドの定義

## スクリプト処理フロー

### 1. 画面設計書からのデータ抽出

画面設計書の「DB登録値」シートから以下の情報を抽出：

- **テーブル名** : セクション区切り行から識別
  - 例: 「制作見積」「制作見積依頼」

- **データ行** : No. が番号の行から抽出
  - **B列** : No. (行番号)
  - **G列** : 項目名 (item name)
  - **O列** : フィールド名 (field name / physical name)
  - **Y列** : その他条件 (other conditions)
  - **AN列** : 登録値 (registration value)

**データ構造例：**
```
テーブル: 制作見積
  No. 1 | 項目名: ステータス      | フィールド名: prod_quot_status
  No. 2 | 項目名: 更新日          | フィールド名: updated_at

テーブル: 制作見積依頼
  No. 1 | 項目名: 制作見積依頼id  | フィールド名: prod_quot_request_id
  No. 2 | 項目名: 制作見積id      | フィールド名: prod_quot_id
  ...
```

### 2. テーブル定義書からのデータ抽出

各テーブルシートから以下の情報を抽出：

- **テーブル名** : シート名
  - 例: 制作見積, 制作見積依頼, 制作見積詳細 等

- **フィールド定義** : ヘッダー行の下からデータ行を抽出
  - **B列** : No. (フィールド番号)
  - **E列** : 論理名 (logical name)
  - **N列** : 物理名 (physical name / field name)

**データ構造例：**
```
シート: 制作見積
  No. 1 | 論理名: 制作見積id      | 物理名: prod_quot_id
  No. 2 | 論理名: 見積id          | 物理名: quot_id
  ...

シート: 制作見積依頼
  No. 1 | 論理名: 制作見積依頼id  | 物理名: prod_quot_request_id
  ...
```

### 3. クロスチェック処理

画面設計書で定義されたフィールドが、対応するテーブルのテーブル定義書に存在するか確認：

1. **テーブル名が存在するか確認**
   - 画面設計書のテーブル名がテーブル定義書のシート名に存在？

2. **フィールド名が存在するか確認**
   - テーブル定義書の物理名（N列）にフィールド名が存在？

**チェック結果**:
- **OK** : フィールドが正しく存在
- **警告** : テーブルまたはフィールドが見つからない

### 4. 結果出力

標準出力およびマークダウンファイルに結果を出力：

**出力ファイル**: `check_result.md`

**マークダウンテーブル形式**:
```
| DB登録値セクション | 行番号 | 項目名 | フィールド名 | チェック結果 |
|---|---|---|---|---|
| 制作見積 | 1 | ステータス | prod_quot_status | OK |
```

## スクリプト内部構造

### 主要関数

#### `extract_screen_design_data() -> List[Dict]`
- 画面設計書の「DB登録値」シートからデータを抽出
- テーブル名、行番号、項目名、フィールド名を取得
- 戻り値: フィールド定義のリスト

#### `extract_table_definitions() -> Dict[str, List[str]]`
- テーブル定義書の全シートからフィールド名を抽出
- 戻り値: テーブル名をキー、フィールドリストを値とする辞書

#### `cross_check(screen_data, table_fields) -> Tuple[List[Dict], List[str]]`
- 画面設計書のフィールドがテーブル定義書に存在するか確認
- 戻り値: (チェック結果リスト, エラーメッセージリスト)

#### `format_markdown_table(results) -> str`
- チェック結果をマークダウンテーブル形式でフォーマット
- 戻り値: マークダウン形式の文字列

## トラブルシューティング

### エラー: ModuleNotFoundError: No module named 'pandas'

```bash
pip install pandas openpyxl
```

### 警告: フィールドが見つからない

1. テーブル定義書でフィールド名（物理名）のスペル確認
2. 画面設計書で項目名やフィールド名のスペル確認
3. テーブル名が正確に対応しているか確認

### チェック結果が0件の場合

- 画面設計書の「DB登録値」シートの構造が想定と異なる可能性
- 列のオフセットを確認してスクリプトを調整
