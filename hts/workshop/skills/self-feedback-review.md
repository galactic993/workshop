# セルフフィードバックループ — 設計書レビューのダブルチェック

前回のレビュー結果（design-review / cross-review / batch-review）を元データ（Excel）と照合し、検証を行ってください。

## 概要

```
1回目: レビュー実行 → 結果出力
2回目: ↓ 本スキル（ダブルチェック）
       結果を元Excelと照合 → 誤検知・見落としを検出
3回目: 修正レビュー結果を出力 → 最終確認
```

> **ルール**: ループは最大3回まで。最後の判断は必ず人間が行う。

---

## ステップ1: 前回のレビュー結果を解析

ユーザーから提供された前回のレビュー結果（Markdownテーブル形式）を解析し、以下を抽出してください：

1. **チェック項目ごとの指摘一覧**（チェックID、行番号、対象値、判定結果）
2. **error / warning / info の件数**
3. **「OK」と判定された項目の一覧**

---

## ステップ2: 元データとの再照合（誤検知チェック）

openpyxl で元の Excel ファイルを読み込み、前回の指摘を**1件ずつ**再検証してください。

### A系チェックの再検証

| チェック | 再検証方法 |
|---------|-----------|
| A1: イベント番号連番 | イベント記述書シートのB列（5行目〜）を直接読み取り、前回指摘の抜け番・重複が実在するか確認 |
| A2: EVENT番号クロスシート | 項目記述書Y列のEVENT番号を取得し、イベント記述書B列と突合。前回「存在しない」とされた番号が本当に存在しないか確認 |
| A3: 種別とイベント矛盾 | 項目記述書のR列（種別）とY列（EVENT No.）を直接確認。前回指摘の行で種別が本当に「ボタン/リンク/プルダウン/ラジオボタン」か、EVENT No.が本当に空欄か確認 |
| A4: 種別とバリデーション矛盾 | 項目記述書のR列とAI列を直接確認。前回指摘の行でデータが正しいか確認 |
| A6: 項目番号連番 | 項目記述書のB列を直接読み取り、前回指摘の抜け番・重複が実在するか確認 |
| A7: メッセージ番号連番 | メッセージ一覧のB列（5行目〜）を直接読み取り、前回指摘の抜け番・重複が実在するか確認 |

### B系チェックの再検証（テーブル定義書がある場合）

| チェック | 再検証方法 |
|---------|-----------|
| B1: DB登録値フィールド名 | DB登録値シートのO列のフィールド名を再取得し、テーブル定義書の対応シートN列と再突合 |
| B2: 参照仕様テーブル.カラム | 参照仕様シートのパターンを再抽出し、テーブル定義書と再突合 |

### C系チェックの再検証（テーブル定義書チェックの場合）

| チェック | 再検証方法 |
|---------|-----------|
| C1: FK参照先存在 | AT列（備考）からFK参照先を再抽出し、対応シートの存在とN列を再確認 |
| C2: PK定義 | 各テーブルシートのAI列にPKフラグが本当にないか再確認 |
| C3: NOT NULL+デフォルト未設定 | AF列とAO列を直接確認。PK/FK除外ロジックが正しく適用されているか確認 |

### 再検証で確認すること

各指摘に対して以下を判定：

- **✅ 確認済**: 元データと一致。指摘は正しい
- **⚠️ 誤検知**: 元データと不一致。指摘は誤り（false positive）
  - 原因を特定: ヘッダー行の誤認識 / セル結合の読み飛ばし / 空白文字の差異 / データ範囲の誤り 等
- **❓ 要確認**: 元データが曖昧で機械的に判断できない（人間の確認が必要）

---

## ステップ3: 見落としチェック（false negative 検出）

前回のレビューで **「OK」と判定された項目** や **チェック対象外になっていた行** を再スキャンし、見落としがないか確認してください。

### 見落としの典型パターン

| パターン | 確認方法 |
|---------|---------|
| セル結合による行スキップ | openpyxl の `merged_cells` を確認し、結合セル内のデータが読み取れているか検証 |
| ヘッダー行の誤認識 | 想定ヘッダー行（4行目 or 5行目）に本当にヘッダーがあるか確認 |
| シート名の表記揺れ | 「イベント記述書」vs「イベント記述」など、部分一致で見落としたシートがないか確認 |
| 空白文字の差異 | 全角スペース、タブ、改行を含む値が正しく比較されているか確認 |
| データ範囲の誤り | 最終行の判定が正しいか確認。空行の後にデータが続いていないか確認 |
| 複数行にまたがるセル | B列に番号がない行（前の項目の続き）が正しく処理されているか確認 |

## ステップ4: 検証レポート出力

以下の形式で検証レポートを出力してください。

```

## 対象ファイル
- レビュー対象: {ファイル名}
- テーブル定義書: {ファイル名}（あれば）

## 検証サマリー

| 区分 | 件数 | 内訳 |
|------|------|------|
| 前回の指摘総数 | {N}件 | error: {n}, warning: {n}, info: {n} |
| ✅ 確認済（正しい指摘） | {N}件 | |
| ⚠️ 誤検知（取り消し） | {N}件 | |
| ❓ 要確認（人間判断必要） | {N}件 | |
| 🔍 新規発見（見落とし） | {N}件 | error: {n}, warning: {n} |

## 信頼度スコア
- 前回レビューの精度: {確認済 / (確認済 + 誤検知) * 100}%
- 網羅性: {確認済 / (確認済 + 新規発見) * 100}%

## 詳細: 誤検知（取り消すべき指摘）

| 1 | A2 | 15 | EVENT0099が存在しない | EVENT0099はB列8行目に存在 | セル結合で読み飛ばし |

| # | チェックID | 行番号 | 発見内容 | 重要度 | 見落としの原因 |
|---|-----------|--------|---------|--------|--------------|
| 1 | A3 | 42 | プルダウンにEVENT番号なし | error | セル結合行で種別を認識できず |

## 詳細: 要確認（人間の判断が必要）

| # | チェックID | 行番号 | 内容 | 判断ポイント |

| 1 | A4 | 30 | テキストインプットに制御なし | 意図的に制御不要の可能性あり |

---

## 修正後レビュー結果（最終版）

（誤検知を除外し、新規発見を追加した修正版のレビュー結果をここに出力）

| チェック | 結果 | error数 | warning数 |
|---------|------|---------|-----------|

## 次のアクション
- [ ] 要確認項目の人間レビュー
- [ ] 修正後レビュー結果の承認
```

## 実行上の注意

1. **openpyxl で直接読み取る**: 前回レビューの記憶や推測ではなく、必ず元 Excel を再読み込みして照合すること
2. **1件ずつ検証**: まとめて判定せず、各指摘を個別に確認すること
3. **原因を特定**: 誤検知や見落としが見つかった場合、なぜそうなったかの原因を記録すること
4. **3回ルール**: このフィードバックループは最大3回まで。3回目で解消しない問題は人間に委ねること
5. **セル結合に注意**: openpyxl でセル結合を正しく処理しているか必ず確認すること。`worksheet.merged_cells.ranges` で結合範囲を取得し、結合セルの値は左上セルから読み取ること
