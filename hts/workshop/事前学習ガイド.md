## 1. LLMとは

### 3行で説明

LLM（Large Language Model）は、大量のテキストデータから言語のパターンを学習したAIです。文章を理解し、質問に答え、文章を生成することができます。ChatGPTやClaudeがその代表的な例です。

### 身近な例

みなさんが日常で使っているものの中にも、LLMを活用したサービスがあります：

- **ChatGPT**（OpenAI） - チャットで会話できるAI
- **Claude**（Anthropic） - 安全性を重視したAIアシスタント
- **Gemini**（Google） - Googleが提供する生成AI
- **Copilot**（Microsoft） - WindowsやOfficeに統合されたAI

### わかりやすい比喩：予測変換の超高度版

LLMを一言で言うと、「スマホの予測変換の超高度版」です。

スマホで「おつかれ」と入力すると、「お疲れ様です」「お疲れ様でした」などが候補に出ますよね。LLMも基本的には同じ仕組みですが、以下の点が段違いです：

- **学習量が膨大** - インターネット上の膨大なテキストを学習
- **文脈を理解** - 会話の流れや文章全体の意味を把握
- **複雑な推論が可能** - 単語の予測だけでなく、論理的な思考もできる

### できること

- 文章の要約・翻訳・校正
- プログラミングコードの生成・解説
- アイデア出し・ブレインストーミング
- データの分析・分類
- 質問への回答

### できないこと・注意点

- **リアルタイム情報は限定的** - 学習データに含まれない最新情報は知らない
- **計算は苦手** - 複雑な計算は間違えることがある
- **幻覚（ハルシネーション）** - もっともらしい嘘をつくことがある
  - 例：存在しない参考文献を作り上げる
  - **対策**: 重要な情報は必ず人間が確認する

### プロンプト（指示文）とは

プロンプトは、AIへの「指示書」や「質問文」のことです。

```
悪いプロンプト例：
「コードを書いて」

良いプロンプト例：
「FileMakerで顧客管理データベースを作成したいです。
以下の要件を満たすスクリプトを書いてください：
- 顧客名、電話番号、メールアドレスを登録
- 重複チェック機能付き
- エラーハンドリング含む」
```

**ポイント**: 具体的に指示するほど、良い結果が得られます。

**なぜ重要か**: プロンプトの書き方ひとつで、AIの出力品質が大きく変わります。研修では、効果的なプロンプトの書き方も学びます。

## 2. コーディングエージェントは何ができるか

### 従来のAIチャットとの違い

| 従来のAIチャット | コーディングエージェント |
|-----------------|------------------------|
| 質問に答えるだけ | ファイルの読み書きができる |
| コード例を示すだけ | 実際にファイルを編集できる |
| コマンドを教えるだけ | コマンドを実行できる |
| 人間がコピペして実行 | エージェントが自律的に実行 |

### エージェントとは

エージェントは、「自律的にファイル操作・コマンド実行ができるAI」です。

従来のAIは「どうすればいい？」という質問に答えるだけでした。エージェントは、指示を受けると、実際に手を動かして作業を完了させます。

```
従来のAI:
ユーザー: 「バグを修正したい」
AI: 「このファイルの〇行目を△△に変更してください」
ユーザー: （自分でファイルを開いて修正）

コーディングエージェント:
ユーザー: 「バグを修正して」
エージェント: （ファイルを読む）→（原因を特定）→（修正）→（テスト実行）→「修正完了しました」
```

### 具体的にできること

- **コード生成**: 要件から自動的にコードを作成
- **バグ修正**: エラーメッセージから原因を特定し、修正
- **テスト作成**: テストコードを自動生成
- **リファクタリング**: コードを整理・最適化
- **ドキュメント作成**: コードの説明書を自動作成
- **コードレビュー**: 品質チェックと改善提案

### 人間の役割の変化

AI時代の開発では、人間の役割が変わります：

| 従来 | AI時代 |
|------|--------|
| コードを書く | 何を作るか指示する |
| デバッグする | AIの出力をレビューする |
| テストを書く | 品質を判断する |
| ドキュメントを書く | 方針を決定する |

**ポイント**: 人間は「指揮官」になり、AIが「実行者」になります。

**なぜ重要か**: AIがコードを書くからといって、人間の責任が減るわけではありません。AIが書いたコードが正しいか、セキュリティ上問題ないかを判断するのは人間の仕事です。「AIが書いたコードを人間がレビューする」という新しいワークフローを理解することが重要です。

## 3. Claude Codeとは

### 概要

Claude Codeは、Anthropic社が提供するCLI（コマンドライン）ベースのコーディングエージェントです。

ターミナル（黒い画面に文字を入力するアプリ）から直接使い、ファイルの読み書きやコマンド実行が可能です。

### 特徴

- **コンテキストウィンドウが大きい**: 長いファイルや複数ファイルを一度に理解できる
- **ツール使用が得意**: ファイル操作、コマンド実行、検索など多機能
- **安全性重視**: 危険な操作前には確認を求める
- **エージェントチーム**: 複数のAIが並列で作業可能

### 今回の研修での使い方

1. **設計書のMarkdown化**: Excelで作成した設計書を、AIが読みやすいMarkdown形式に変換
2. **開発作業の自動化**: 設計書に基づいて、スクリプトやフィールド定義を自動生成
3. **テストコードの作成**: 動作確認用のテストを自動作成

**ポイント**: Claude Codeは「チャットで相談する」ツールではなく、「作業を任せる」ツールです。

**なぜ重要か**: 従来、人間が何時間もかかっていた作業を、Claude Codeに任せることで大幅に短縮できます。ただし、AIへの指示の出し方（プロンプトエンジニアリング）が成果を左右します。

## 4. Claude in Excelとは

### 概要

Claude in Excelは、Excel上でClaudeを使えるアドイン（拡張機能）です。

Excelのセルの中で、関数のようにAIを呼び出すことができます。

### 使い方のイメージ

従来のExcel関数：
```
=SUM(A1:A10)
```

Claude in Excel：
```
=CLAUDE("この設計書の不整合を指摘してください", A1:D10)
```

### 今回の研修での使い方

- **設計書のレビュー自動化**: 整合性チェック、漏れの検出
- **説明文の生成**: フィールド定義から説明文を自動作成
- **変換処理**: データ形式の変換

### メリット

- **大量のセルを一括処理**: 数百行のデータに対して一括でAI処理
- **Excelから離れずにAI活用**: コピー＆ペースト不要
- **既存ワークフローとの統合**: 慣れ親しんだExcelでAIを使える

**ポイント**: Claude in Excelは「Excelの機能をAIで拡張する」ツールです。

**なぜ重要か**: 設計書作成はExcelで行うことが多く、Claude in Excelを使うことで、Excel内で完結した効率的なレビューが可能になります。

## 5. Claude Codeのインストール

### 前提条件

- **Node.js（v18以上）**: JavaScriptを実行するための環境
- **ターミナルの基本操作**: コマンドを入力して実行できること

#### Node.jsがインストールされているか確認

ターミナル（Macの場合）またはコマンドプロンプト（Windowsの場合）を開き、以下を入力：

```bash
node --version
```

`v18.x.x` 以上のバージョンが表示されればOKです。

#### Node.jsのインストール（未インストールの場合）

1. https://nodejs.org/ にアクセス
2. LTS（推奨版）をダウンロード
3. インストーラーの指示に従ってインストール

### インストール手順

#### Step 1: Claude Codeをインストール

ターミナルで以下のコマンドを入力：

```bash
npm install -g @anthropic-ai/claude-code
```

※ `npm` は Node.js に付属するパッケージマネージャーです

#### Step 2: Anthropicアカウントの作成

1. https://console.anthropic.com/ にアクセス
2. 「Sign Up」をクリック
3. メールアドレスとパスワードを入力
4. メール認証を完了

#### Step 3: APIキーの取得

1. https://console.anthropic.com/ にログイン
2. 「API Keys」をクリック
3. 「Create Key」をクリック
4. キーに名前を付けて作成
5. **重要**: 表示されたキーをコピーして安全な場所に保存
   - キーは二度と表示されません

#### Step 4: 支払い方法の設定

Claude Codeは従量課金制です：

1. https://console.anthropic.com/settings/billing にアクセス
2. クレジットカード情報を入力
3. 使用上限を設定（オプション）

#### Step 5: 初回起動と認証

ターミナルで以下を入力：

```bash
claude
```

初回起動時に認証フローが始まります：

1. 表示されるURLをブラウザで開く
2. Anthropicアカウントでログイン
3. 認証コードをコピー
4. ターミナルに貼り付け

### 動作確認

認証が完了したら、Claude Codeが正常に動作するか確認します。

Claude Codeのプロンプト（`>` が表示されている状態）で、以下のように入力：

```
こんにちは。自己紹介してください。
```

Claudeからの応答があれば、インストール成功です。

終了するには `/exit` と入力します。

**ポイント**: APIキーは他人に教えないでください。クレジットカードと同じくらい重要な情報です。

**なぜ重要か**: Claude Codeは研修での中心的なツールです。事前にインストールを完了させておくことで、研修当日は学習に集中できます。

## 6. Claude in Excelのインストール

### 前提条件

- **Microsoft 365のExcel（デスクトップ版）**
  - Web版（Excel Online）では動作しません
  - Microsoft 365のサブスクリプションが必要です

### インストール手順

#### Step 1: Excelを開く

Excelを起動し、新しいブックを作成します。

#### Step 2: アドインストアを開く

1. 「挿入」タブをクリック
2. 「アドインを取得」または「Office アドイン」をクリック

#### Step 3: Claude for Excelを検索・インストール

1. 検索ボックスに「Claude」と入力
2. 「Claude for Excel」を選択
3. 「追加」をクリック

※ 見つからない場合は、以下の方法もあります：
- https://appsource.microsoft.com にアクセス
- 「Claude for Excel」を検索してインストール

#### Step 4: APIキーの設定

1. Excelのリボンに「Claude」タブが追加されます
2. 「Claude」タブをクリック
3. 「Settings」または「設定」をクリック
4. Anthropic APIキーを入力
5. 「Save」をクリック

※ APIキーは Claude Code と同じものが使用できます

### 動作確認

任意のセルに以下の数式を入力：

```
=CLAUDE("こんにちは")
```

Claudeからの挨拶が返ってくれば、インストール成功です。

### 基本的な使い方

```
=CLAUDE("質問や指示")
=CLAUDE("このテキストを要約して", A1)
=CLAUDE("このデータの不整合を指摘して", A1:D10)
```

**ポイント**: Claude in Excelの使い方は、研修の中で詳しく学びます。まずはインストールと基本的な動作確認を済ませておいてください。

**なぜ重要か**: 研修では設計書のレビューにClaude in Excelを使用します。事前に慣れておくことで、研修当日の作業がスムーズになります。

## 補足セクション

### よくある質問（FAQ）

#### Q1. 費用はどのくらいかかりますか？

**A.** API使用料は従量課金制です。

- Claude Code: 1会話あたり数円〜数十円程度（作業内容による）
- Claude in Excel: セルごとの課金、1セルあたり〜1円程度

研修期間中の目安: 1,000〜3,000円程度

**注意**: 使用量を監視し、予算上限を設定することをお勧めします。

#### Q2. 会社の情報を入力して大丈夫ですか？

**A.** API経由の場合、Anthropicは入力データをモデルの学習に使用しません。

ただし、以下に注意してください：
- 機密情報（パスワード、個人情報など）は入力しない
- 会社のセキュリティポリシーを確認する
- 不安な場合は上司に相談する

#### Q3. プログラミングできなくても使えますか？

**A.** はい、使えます。

- Claude Code: プログラミング知識がなくても、自然言語で指示できます
- Claude in Excel: Excel関数の知識があれば使えます

研修では、必要な知識を一から学びます。

#### Q4. インストールがうまくいかない場合は？

**A.** 以下を確認してください：

1. **Node.jsのバージョン**: `node --version` でv18以上か確認
2. **管理者権限**: Windowsの場合、管理者として実行
3. **ネットワーク**: 会社のファイアウォールがブロックしていないか確認
4. **エラーメッセージ**: エラー内容を記録して研修時に相談

それでも解決しない場合は、研修当日の開始前に講師に相談してください。

#### Q5. 事前にどの程度使い慣れておくべきですか？

- [ ] Claude in Excelのインストール完了
- [ ] どちらも動作確認済み
