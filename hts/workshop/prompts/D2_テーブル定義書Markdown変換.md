# D2: テーブル定義書 → Markdown変換

**対象ファイル:** テーブル定義書（例: `編-テーブル定義書_1012.xlsx`）

## プロンプト

```
このブックに含まれる全シートを読み取り、1つのMarkdownドキュメントとして出力してください。
出力はそのままコピペして .md ファイルとして保存できる形式にしてください。

以下のルールに従ってください：

■ 全体構造
- 最初に表紙の情報（システム名、区分、作成日、変更日、作成者、変更者）をYAMLフロントマター風に出力
- 「表紙」「変更履歴」シートは出力に含めない

■ テーブル一覧シート
- ## テーブル一覧（h2見出し）
- ヘッダー: B列=No., E列=論理名, P列=物理名, AA列=種別, AG列=概要
- Markdownテーブルとして出力
- 列: No. | 論理名 | 物理名 | 種別 | 概要

■ 各テーブル定義シート（テーブル一覧以外のデータシート）
- ## テーブル名（h2見出し）。形式: 「## 論理名 (`物理名`)」
- ヘッダー行(1-2行目)からテーブルのメタ情報を出力:
  - 論理名: U1セル
  - 物理名: U2セル
  - 作成日/変更日/作成者/変更者
- ヘッダー行5: B列=No., E列=論理名, N列=物理名, W列=データ型, AC列=桁数, AF列=Not Null, AI列=PK, AK列=UK, AM列=FK, AO列=Default, AT列=備考
- カラム定義をMarkdownテーブルで出力
- 列: No. | 論理名 | 物理名 | データ型 | 桁数 | Not Null | PK | UK | FK | Default | 備考
- 「●」は ✔ に変換して出力
- 空セルは - で埋める
- データ行の後に補足説明（★で始まるセクション等）がある場合は、blockquote（> ）として出力

すべての出力をMarkdownのコードブロックなしで、そのままMarkdown文書として出力してください。
```

## 説明

このプロンプトは、テーブル定義書のすべてのシートを1つのMarkdownドキュメントに変換します。

- **用途**: テーブル定義書をMarkdown形式に変換して、Claude Codeや他のAIツールに読み込ませる
- **出力形式**: そのままコピペで `.md` ファイルとして保存可能

## 使用上の注意

- テーブル定義書のみを開いた状態で実行してください
- 各シートのヘッダー行番号が記載されている通りであることを確認してください
- テーブル一覧シートが存在することを確認してください
- ヘッダー行が異なる場合は、プロンプト内の行番号を修正してください
- 「●」などの特殊文字が正しく変換されることを確認してください
