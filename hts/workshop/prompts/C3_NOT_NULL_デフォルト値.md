# C3: NOT NULL かつデフォルト値未設定チェック

**重要度:** info
**対象シート:** テーブル定義書の全テーブルシート

## プロンプト

```
テーブル定義書の全テーブルについて、NOT NULL制約があるのにデフォルト値が設定されていないカラムをチェックしてください。

【各テーブルシートの構造】
- AF列=Not Null（Y/Nや○/×など）
- AI列=PK（主キーフラグ）
- AM列=FK（外部キーフラグ）
- AO列=Default（デフォルト値）

チェック内容：
1. AF列（Not Null）が「Y」または「○」または「YES」の行を抽出
2. 同時に以下の条件を除外：
   - AI列（PK）に値がある行（PKは自動採番されることが多い）
   - AM列（FK）に値がある行（FKは親テーブルの値を参照）
3. AO列（Default）が空欄の行を検出

結果を以下の形式で出力してください：
| シート名 | 行番号 | カラム名（論理） | カラム名（物理） | チェック結果 |
|----------|--------|------------------|------------------|--------------|
例：| 顧客 | 5 | 氏名 | customer_name | 情報: NOT NULLだがデフォルト値なし |
    | 顧客 | 12 | ステータス | status | OK（デフォルト値: 0） |
```

## 説明

このプロンプトは、テーブル定義書のカラム定義について、NOT NULL制約のあるカラムにデフォルト値が設定されているかをチェックします。

- **対象**: NOT NULL制約のあるカラム（PKとFKを除く）
- **チェック内容**: デフォルト値が設定されているか確認
- **重要性**: NOT NULLカラムにデフォルト値がないと、レコード挿入時に値を必ず指定する必要があります

## 使用上の注意

- このチェックは「情報」レベルの重要度です。エラーではなく、設計上の注意です
- PKは自動採番されることが多いため、デフォルト値がなくても問題ありません
- FKは親テーブルの値を参照するため、デフォルト値の設定パターンが異なります
- NOT NULLカラムに適切なデフォルト値を設定することで、アプリケーション側の処理が簡潔になります
- ステータスや区分値などは、デフォルト値を設定することを推奨します
