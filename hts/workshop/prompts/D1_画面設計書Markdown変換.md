# D1: 画面設計書 → Markdown変換

**対象ファイル:** 画面設計書（例: `02-02_2_編-制作見積-制作見積書作成トップ_画面設計書_1007.xlsx`）

## プロンプト

```
このブックに含まれる全シートを読み取り、1つのMarkdownドキュメントとして出力してください。
出力はそのままコピペして .md ファイルとして保存できる形式にしてください。

以下のルールに従ってください：

■ 全体構造
- 最初に表紙の情報（システム名、サブシステム名、画面名、作成日、変更日、作成者、変更者）をYAMLフロントマター風に出力
- 各シートは ## 見出し（h2）で区切る
- シートの出力順序: 画面概要 → イベント記述書 → 項目記述書 → 参照仕様 → DB登録値 → 表示切替項目 → メッセージ一覧
- 「表紙」「変更履歴」「Sheet1」など空シートは出力に含めない

■ 画面概要シート
- 画面の説明文をそのまま出力
- 「利用権限一覧」はMarkdownテーブルで出力（権限種別 | 利用機能 | 備考）
- 「モジュール一覧」はMarkdownテーブルで出力（モジュール名 | モジュールパス | 内容）
- 「I/O」セクションはフローを箇条書きで出力

■ イベント記述書シート
- ヘッダー行5: B列=EVENT No., G列=トリガー, U列=区分, Z列=処理内容
- 各EVENTを ### 見出し（h3）で区切る（例: ### EVENT0001: 初期表示時）
- 区分を **太字** で表示
- 処理内容はインデント付きの番号リストで出力。サブ項目は入れ子にする
- EVENTごとにセクション内の複数行を結合して出力すること（B列に次のEVENT番号が出るまでが1つのEVENTの範囲）

■ 項目記述書シート
- ヘッダー行4: B列=No., G列=項目名, R列=種別, Y列=EVENT No., AD列=制御区分, AI列=制御内容, AS列=メッセージ
- Markdownテーブルとして出力
- 列: No. | 項目名 | 種別 | EVENT No. | 制御区分 | 制御内容 | メッセージ
- 1つの項目が複数行にまたがる場合（B列に番号がなくAI列やAS列に値がある行）は、前の項目の制御内容・メッセージに改行区切りで結合する

■ 参照仕様シート
- 自由形式テキストのため、見出し（◆で始まる行）を ### h3 で出力
- 番号付き手順（1., 2. 等）を番号リストで出力
- <抽出テーブル>, <抽出項目>, <抽出条件> はそれぞれ **太字ラベル** として出力
- 条件分岐（「〜の場合」）はインデント付き箇条書きで出力

■ DB登録値シート
- テーブルごとにセクション分けされている（B列にテーブル名、その下にヘッダー行: B列=No., G列=項目名, O列=フィールド名, Y列=その他条件, AN列=登録値）
- テーブル名を ### h3 見出しで出力
- 各テーブルの内容をMarkdownテーブルで出力
- 列: No. | 項目名 | フィールド名 | その他条件 | 登録値

■ 表示切替項目シート
- ヘッダー行5: B列=共通表示項目, P列=条件, AD列=追加表示項目, AR列=備考
- 条件ごとにセクション分けして出力
- 共通表示項目は冒頭で箇条書き
- 各条件とそれに対応する追加表示項目を箇条書きで出力

■ メッセージ一覧シート
- ヘッダー行5: B列=No., G列=条件, U列=区分, AD列=メッセージ
- Markdownテーブルとして出力
- 列: No. | 条件 | 区分 | メッセージ
- セルが複数行にまたがる場合（B列に番号がなくG列に値がある行）は前の行に結合

すべての出力をMarkdownのコードブロックなしで、そのままMarkdown文書として出力してください。
```

## 説明

このプロンプトは、画面設計書のすべてのシートを1つのMarkdownドキュメントに変換します。

- **用途**: 画面設計書をMarkdown形式に変換して、Claude Codeや他のAIツールに読み込ませる
- **出力形式**: そのままコピペで `.md` ファイルとして保存可能

## 使用上の注意

- 画面設計書のみを開いた状態で実行してください
- 各シートのヘッダー行番号が記載されている通りであることを確認してください
- ヘッダー行が異なる場合は、プロンプト内の行番号を修正してください
- 出力が長い場合は、セクションごとに実行することも検討してください
