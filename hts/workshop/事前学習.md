# 事前学習ガイド

## コーディングエージェントとは？

AIがタスクを受け取り、**計画→コード編集→コマンド実行→検証**まで自律的にこなす開発支援ツール。

| 世代 | 例 | できること |
|------|----|-----------|
| コード補完 | Copilot初期 | カーソル位置の次の行を予測 |
| チャット | ChatGPT | 質問に回答。実行はしない |
| **エージェント** | **Claude Code等** | **複数ファイルを横断して自律的に実装・テスト** |

## リサーチ性能について

- OpenAIのThinkingモード / Proモード、およびGrokは、実際にブラウザを操作しながら情報収集できる
- 通常の情報収集ではスクレイピングを使うことが多いが、スクレイピング対策されたサイトでは必要な情報を取りにくいことがある
- これらのモードはブラウザ経由でページ遷移や画面操作を行えるため、一般的なAIでは取得しにくい情報まで到達しやすく、リサーチ用途に向いている
- OpenAIにはエージェントモードもあり、ブラウザ経由でサイト操作が可能。ただし、現時点ではやや不安定な場面がある

### ブラウザ操作をしたい場合

- Claude CodeやCodexのようなコーディングエージェントも、別途プラグインや外部連携を入れることでブラウザ操作が可能になる
- 調査や検証でブラウザを触らせたい場合は、この拡張経由のブラウザ操作を基本の推奨手段とする

## セキュリティ

### データの再学習利用

| プラン | 学習利用 | 備考 |
|--------|---------|------|
| 無料 | **される可能性あり** | 設定でOFF可の場合も |
| 有料個人 | **設定次第** | OFFにすれば使われない |
| API / Team / Enterprise | **されない** | 商用規約で明確に禁止 |

### シークレット入力について

**結論: 有料/APIプランなら基本OK。ただしリスクゼロではない**

- 有料/APIではデータは再学習に使われない。通信はTLS暗号化
- 残るリスク: サーバーへの一時送信、万が一の漏洩、社内ポリシーとの整合
- **ベストプラクティス**: `.env` + `.gitignore` で管理。本番シークレットは避ける

## 今回使用するエージェント: Claude Code

### 選定理由
- 機能が最も豊富（MCP、サブエージェント、フック、スキル等）
- Excel操作にも対応（Claude in Excel）
- 日本語ドキュメントあり

### 主要機能

| 機能 | 内容 |
|------|------|
| Permission | 操作の承認制御（読み取り/書き込み/コマンド） |
| MCP | 外部ツール連携（Slack, Jira, DB等） |
| Skills | カスタムコマンド（チーム共有可） |
| Hooks | 自動フォーマット、リント等の自動実行 |
| Agents | サブエージェントで並列実行 |
| CLAUDE.md | プロジェクト固有ルールをMarkdownで定義 |
