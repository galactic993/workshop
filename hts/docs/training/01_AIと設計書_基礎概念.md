## 1. AI（LLM）の基本

### LLMとは
- 大量のテキストデータから言語パターンを学習したモデル
- 入力テキストに対して、最も確率の高い続きを生成
- プログラミング、翻訳、分析など多用途に活用可能

### LLMの得意・不得意

| 得意 | 不得意 |
|------|--------|
| テキストの構造化・整理 | 正確な数値計算 |
| パターン認識・異常検出 | 最新情報の参照 |
| コード生成・レビュー | 長い文脈の完全な記憶 |
| 文書の要約・変換 | 画像内の文字認識(OCR) |

### 開発における活用シーン
- 設計書の整合性チェック
- コードの自動生成（マイグレーション、モデル等）
- テストコードの生成
- ドキュメントの自動生成

## 2. なぜExcel設計書をAI可読形式に変換するのか

### Excel設計書の課題
- セル結合、書式情報が多く、テキストとして読みにくい
- AIに直接Excel（バイナリ）を渡せない
- 人間が目視確認するしかなく、チェック漏れが発生

### AI可読形式（Markdown/JSON）への変換メリット
1. **AIが直接理解可能**: テキスト形式でLLMに入力できる
2. **自動チェック可能**: プログラムで整合性を検証できる
3. **バージョン管理可能**: Gitで差分管理できる
4. **再利用性向上**: コード生成の入力として使える

### 変換パイプラインの概要
```
Excel設計書 → Pythonパーサー → 構造化データ(dict) → Markdown/JSON
                                     ↓
                              自動レビューツール → レビューレポート
                                     ↓
                              AI(LLM)入力 → コード生成等
```

## 3. Markdown形式とJSON形式

### Markdown
- 軽量マークアップ言語
- 見出し(`#`)、表(`|`)、リスト(`-`)で構造化
- GitHubやRedmineで標準的に使用

### JSON
- データ交換フォーマット
- プログラムから直接利用しやすい
- APIとの連携に最適

### 使い分け
| 用途 | 推奨形式 |
|------|----------|
| 人間が読む資料 | Markdown |
| プログラムへの入力 | JSON |
| AIへの入力 | どちらもOK（Markdownが自然） |
| Git管理 | どちらもOK |
