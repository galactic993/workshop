# AI活用設計書処理 研修カリキュラム

## 研修概要

- **対象者**: 管理会計システム開発チームメンバー
- **前提知識**: Python基礎、Excel設計書の読み方
- **全5回**: 各回2時間（講義1時間 + ハンズオン1時間）
- **使用ツール**: Python 3, openpyxl, Claude Code

## カリキュラム一覧

### 第1回: AIと設計書 - 基礎概念
- AI（LLM）の基本的な仕組みと得意・不得意
- なぜExcel設計書をAI可読形式に変換するのか
- Markdown/JSON形式の利点
- ハンズオン: 実際の画面設計書Excelの構造を確認

### 第2回: Excelパーサーの理解と活用
- openpyxlの基本操作
- screen_design_parser.pyの構造解説
- table_def_parser.pyの構造解説
- main.py CLIの使い方
- ハンズオン: 新しい画面設計書をパースしてMarkdownに変換

### 第3回: 設計書自動レビューの仕組み
- レビューチェック項目の設計思想（A1-A7, B1-B2, C1-C3）
- reviewer.pyのコードウォークスルー
- table_reviewer.pyのコードウォークスルー
- ハンズオン: reviewコマンドを実行し、レビュー結果を読み解く

### 第4回: カスタムチェック項目の追加
- 新しいチェックルールの設計方法
- チェック関数のテンプレート
- 正規表現を使ったパターンマッチング
- ハンズオン: 独自のチェック項目を実装して動作確認

### 第5回: AI駆動開発ワークフロー実践
- Claude CodeによるコードGeneration
- 設計書→実装コードの自動生成パイプライン
- プロンプトエンジニアリングの基礎
- ハンズオン: 設計書からLaravelマイグレーション/モデルの自動生成

## 研修の目的

### 主な学習目標
1. **AIによる設計書処理の理解**
   - AIを活用した設計書の自動処理・レビューの実践力を養う
   - 既存のExcel設計書をAI可読形式に変換し、自動チェックする手法を学ぶ

2. **ツールの実践的活用**
   - Phase 1で構築したExcelパーサー（screen_design_parser.py, table_def_parser.py, main.py CLI）の活用
   - Phase 2で構築した自動レビューツール（reviewer.py: A1-A7/B1-B2, table_reviewer.py: C1-C3）の活用

3. **AI駆動開発の可能性と限界の理解**
   - AIによる開発支援の可能性と限界を理解する
   - 効果的なプロンプトエンジニアリングの習得
   - 人間とAIの協働ワークフローの構築

### 期待される成果
- Excel設計書を自動的にMarkdown/JSON形式に変換できる
- 設計書の品質を自動チェックし、問題箇所を特定できる
- カスタムチェックルールを追加・拡張できる
- AI支援により設計書から実装コードを生成できる

## プロジェクト背景

本研修は、管理会計システム開発プロジェクト（Next.js 15 + Laravel 11 + PostgreSQL 15）における実際のツール群を活用した実践的な内容です。

### 既存ツールの構成
- **Phase 1**: Excel設計書パーサー
  - screen_design_parser.py: 画面設計書をMarkdownに変換
  - table_def_parser.py: テーブル定義書をMarkdownに変換
  - main.py: CLI統合インターフェース

- **Phase 2**: 自動レビューツール
  - reviewer.py: 画面設計書レビュー（A1-A7, B1-B2チェック項目）
  - table_reviewer.py: テーブル定義書レビュー（C1-C3チェック項目）

これらのツールを実際に活用しながら、AI駆動開発の実践的なスキルを習得します。
